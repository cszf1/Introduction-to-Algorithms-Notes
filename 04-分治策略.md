## 本章导读

### 🎯 一句话概括
> **分治策略就是"化整为零、各个击破"**——把一个大问题拆成几个小问题，分别解决后再合并结果。

### 🏠 生活类比：分治就像"大扫除"

想象你家要进行大扫除：

| 传统做法（一个做） | 分治策略（分工合作） |
|:--|:--|
| 你一个人擦窗户、扫地、洗厕所... | 把任务分成多个子任务 |
| 效率低、耗时长 | 多人同时工作 |
| 累得半死 | 各司其职，效率翻倍 |
| 干完这个才能干下一个 | 同一时间并行处理多个任务 |

**计算机中的分治策略**：
- **分解（Divide）**：把原问题分成若干个规模较小的子问题
- **解决（Conquer）**：递归地解决每个子问题
- **合并（Combine）**：将子问题的解合并成原问题的解

### 🤔 为什么要学分治策略？

1. **很多问题天生适合分治**：排序、搜索、矩阵乘法、最近点对...
2. **是理解递归的基础**：递归式无处不在
3. **是很多高级算法的前置知识**：动态规划、贪心算法都与此相关
4. **面试必考**：递归式复杂度分析是程序员面试的核心考点

## 4.1 分治策略的核心思想

### 📖 正式定义

> **分治策略（Divide and Conquer）**：一种递归算法范式，将原问题划分为**n个与原问题形式相同但规模更小**的子问题，递归求解这些子问题，然后再将子问题的解**合并**为原问题的解。

### 🔄 分治策略的三个步骤

```
┌─────────────────────────────────────────────────────────────┐
│                      原问题 (规模为n)                        │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼ 分解 (Divide)
          ┌───────────────────┼───────────────────┐
          ▼                   ▼                   ▼
    ┌──────────┐        ┌──────────┐        ┌──────────┐
    │ 子问题1  │        │ 子问题2  │   ...  │ 子问题n  │
    │ (n/k规模)│        │ (n/k规模)│        │ (n/k规模)│
    └─────┬────┘        └─────┬────┘        └─────┬────┘
          │                   │                   │
          ▼ 递归解决          ▼ 递归解决          ▼ 递归解决
    ┌──────────┐        ┌──────────┐        ┌──────────┐
    │ 子问题1  │        │ 子问题2  │   ...  │ 子问题n  │
    │   解     │        │   解     │        │   解     │
    └─────┬────┘        └─────┬────┘        └─────┬────┘
          │                   │                   │
          └───────────────────┼───────────────────┘
                              │ 合并 (Combine)
                              ▼
                    ┌─────────────────┐
                    │    原问题解     │
                    └─────────────────┘
```

### 🎯 分治策略的通用框架

让我们用伪代码来表示分治策略的通用结构：

```
DIVIDE-AND-CONQUER(P):
    if P 的规模足够小:
        return ADHOC(P)  // 基本情况，直接求解
    
    # 分解步骤
    div = split(P)  // 将P分解为子问题
    
    # 解决步骤（递归调用）
    for each subproblem div_i:
        y_i = DIVIDE-AND-CONQUER(div_i)  // 递归解决每个子问题
    
    # 合并步骤
    return COMBINE(y_1, y_2, ..., y_k)  // 合并子问题的解
```

### 💡 关键点解析

1. **什么情况下"足够小"？**
   - 基本情况（Base Case）：问题规模小到可以直接求解
   - 通常是 n = 1（只有一个元素）或某个常数阈值
   - 比如排序1个元素，本身就是有序的，无需任何操作

2. **如何"分解"？**
   - 分解要尽可能**均匀**（平衡）
   - 不均匀的分解会导致效率下降
   - 常见的分解方式：平均分成2份、3份...

3. **如何"合并"？**
   - 这是分治策略的核心复杂度来源
   - 有的算法合并很简单（如归并排序）
   - 有的算法合并很复杂（如矩阵乘法）

## 4.2 递归式的引入

### 📖 什么是递归式？

> **递归式（Recurrence）**：一个等式或不等式，通过更小的输入值来描述一个函数。

### 🔄 递归式与分治策略的关系

分治策略的算法运行时间通常用递归式来描述：

```
T(n) = 
    Θ(1)                                      若 n ≤ n₀（基本情况）
    aT(n/b) + D(n) + C(n)                     若 n > n₀（递归情况）
```

其中：
- **a**：子问题的个数（≥ 1）
- **n/b**：子问题的规模（假设每个子问题规模相同）
- **D(n)**：分解问题的时间
- **C(n)**：合并解的时间

### 📝 归并排序的递归式详解

归并排序（Merge Sort）是分治策略的经典例子：

```
┌────────────────────────────────────────────────────────────┐
│                   归并排序的递归式                          │
│                                                            │
│    T(n) = Θ(1)                    若 n = 1（基本情况）     │
│    T(n) = 2T(n/2) + Θ(n)          若 n > 1（递归情况）     │
│                                                            │
│    解释：                                                  │
│    - 2T(n/2)：把问题分成2个子问题，每个规模为n/2           │
│    - Θ(n)：合并两个已排序数组需要线性时间                   │
└────────────────────────────────────────────────────────────┘
```

### 🎯 递归式的三种情况

| 情况 | 条件 | 结果 | 例子 |
|:---:|:---:|:---:|:---|
| 情况1 | D(n) + C(n) = O(1) | T(n) = Θ(n^(log_b a)) | 归并排序 |
| 情况2 | D(n) + C(n) = Θ(n^d) 且 a = b^d | T(n) = Θ(n^d log n) | 归并排序 (a=2, b=2, d=1) |
| 情况3 | D(n) + C(n) = Θ(n^d) 且 a < b^d | T(n) = Θ(n^d) | - |

> ⚠️ **FAQ**：为什么会有这三种情况？
> 
> 这三种情况描述了子问题数量与分解/合并成本的相对关系：
> - 情况1：分解+合并成本太小，子问题数量决定复杂度
> - 情况2：两者平衡，多出来的 log n 来自递归深度
> - 情况3：分解+合并成本太大，成为瓶颈

## 4.3 递归式求解方法

### 📋 方法总览

求解递归式主要有三种方法：

| 方法 | 适用场景 | 难度 | 特点 |
|:---|:---|:---:|:---|
| **代入法（Substitution）** | 所有递归式 | ⭐⭐⭐ | 猜一个答案，数学归纳法验证 |
| **递归树法（Recursion Tree）** | 直观理解 | ⭐⭐ | 画出递归过程图，帮助理解 |
| **主方法（Master Method）** | 特定形式 | ⭐⭐⭐⭐⭐ | 直接套公式，最实用 |

---

## 4.3.1 代入法（Substitution Method）

### 🎯 核心思想

代入法分为两步：
1. **猜测（Guess）**：猜测解的形式
2. **验证（Verify）**：用数学归纳法证明猜测正确

### 📝 完整示例：证明 T(n) = 2T(n/2) + Θ(n) = O(n log n)

**第一步：猜测**

根据经验，我们猜测 T(n) = O(n log n)。

**第二步：验证（数学归纳法）**

我们假设对于所有 < n 的值，都有 T(k) ≤ c·k·log k（c 是某个待定的正常数）。

证明 T(n) ≤ c·n·log n：

```
T(n) = 2T(n/2) + n
     ≤ 2·c·(n/2)·log(n/2) + n      [使用归纳假设]
     = c·n·log(n/2) + n
     = c·n·(log n - log 2) + n
     = c·n·log n - c·n·1 + n
     = c·n·log n + (1-c)n
     
若 c ≥ 1，则 (1-c)n ≤ 0
所以 T(n) ≤ c·n·log n ✓
```

**第三步：验证基本情况**

对于 n = 2：
```
T(2) ≤ c·2·log 2 = 2c
```
只要 c ≥ T(2)/2，基本情况就成立。

**结论**：T(n) = O(n log n) ✓

### ⚠️ 代入法的常见陷阱

| 陷阱 | 说明 | 解决方法 |
|:---|:---|:---|
| **猜测太松** | 猜 O(n³) 永远是对的，但不精确 | 结合递归树法获得更紧的界 |
| **忽略常数** | 忘记验证基本情况 | 必须验证 n = O(1) 的情况 |
| **代数错误** | 推导过程中算错 | 仔细检查每一步 |
| **过度规范化** | 强行把解凑成猜测形式 | 保持灵活性，必要时调整猜测 |

### 💡 代入法技巧

1. **从递归树获得猜测**：先画递归树，再猜测
2. **验证紧界**：先证上界，再证下界（Ω）
3. **做变量替换**：令 m = log n，简化问题
4. **多猜几次**：第一次猜错了很正常

## 4.3.2 递归树法（Recursion Tree Method）

### 🎯 核心思想

递归树法通过**画图**来可视化递归过程，把递归式展开成一棵树。

### 🌳 递归树的组成部分

```
                    递归树
                    
         根节点：代表原问题 T(n)
              │
              ▼
         ┌─────────┐
         │   T(n)  │  ← 第一层：分解成本 c·n
         └────┬────┘
              │ 划分为 a 个子问题
         ┌────┴────┐
         │         │
      ┌──┴──┐   ┌──┴──┐   ← 第二层：a 个节点，每个 T(n/b)
      │T(n/b)│   │T(n/b)│
      └──┬──┘   └──┬──┘
         │         │
         ▼         ▼
       ...       ...
```

### 📝 完整示例：T(n) = 3T(n/4) + n²

让我们画出递归树：

```
层数          成本                     子问题规模
─────────────────────────────────────────────────
第0层:        n²                          n
             /  |  \
第1层:     n²/4 n²/4 n²/4    (3个)        n/4
           /|   /|   /|
第2层:   n²/16 n²/16 ...      (3²个)      n/4²
          ...
          ↓
直到子问题规模变为 Θ(1)
```

**计算每层的成本**：

| 层数 | 节点数 | 每个节点成本 | 该层总成本 |
|:---:|:---:|:---:|:---:|
| 0 | 1 | n² | n² |
| 1 | 3 | (n/4)² | 3·n²/16 |
| 2 | 9 | (n/4²)² | 9·n²/256 |
| i | 3^i | (n/4^i)² | 3^i · n² / 16^i |

**关键观察**：
- 第 i 层的成本 = 3^i · n² / 16^i = n² · (3/16)^i
- 由于 3/16 < 1，这是一个**几何递减**的序列

**求和**：

```
总成本 = n² + n²·(3/16) + n²·(3/16)² + n²·(3/16)³ + ...
       = n² · Σ[(3/16)^i] (i = 0 到 ∞)
       < n² · 1/(1 - 3/16)    [几何级数求和公式]
       = n² · 16/13
       = O(n²)
```

**结论**：T(n) = O(n²)

### 🎯 递归树的画法步骤

1. **确定根节点**：从递归式出发
2. **确定分支数**：递归式中有几个 T(...)
3. **确定每个分支的规模**：除以 b
4. **确定每层的成本**：D(n) + C(n) 的形式
5. **确定树的高度**：直到叶子节点规模为 Θ(1)
6. **求和**：计算所有层的总成本

### ⚠️ 常见错误

| 错误 | 问题 | 正确做法 |
|:---|:---|:---|
| **忘记减枝** | 树画到无限 | 画到规模为常数 |
| **成本算错** | 每层成本计算错误 | 仔细计算每层的 D(n) + C(n) |
| **级数求和错** | 几何级数公式记错 | 记住：Σr^i = 1/(1-r)（当 |r|<1）|

## 4.3.3 主方法（Master Method）

### 📖 公式定义

> **主定理（Master Theorem）**：设 T(n) = aT(n/b) + f(n)，其中 a ≥ 1，b > 1。
> 
> 那么 T(n) 的渐近复杂度为：

```
情况1（子问题主导）：若 f(n) = O(n^(log_b(a) - ε))，其中 ε > 0
    则 T(n) = Θ(n^(log_b(a)))

情况2（平衡点）：若 f(n) = Θ(n^(log_b(a)) · log^k n)，其中 k ≥ 0
    则 T(n) = Θ(n^(log_b(a)) · log^(k+1) n)

情况3（成本主导）：若 f(n) = Ω(n^(log_b(a) + ε))，其中 ε > 0
    且满足正则条件：af(n/b) ≤ cf(n)，其中 c < 1
    则 T(n) = Θ(f(n))
```

### 📊 主方法的三种情况图解

```
          T(n) = aT(n/b) + f(n)
          
┌────────────────────────────────────────────────────────────────┐
│                                                                │
│   情况1: f(n) 多项式小于 n^(log_b a)                           │
│   ┌──────────────────────────────────────┐                     │
│   │         n^(log_b a)                  │ ← 子问题树的总成本  │
│   │    ████████████████████               │                     │
│   │                      ████████          │ ← f(n)              │
│   │                                     ███│                     │
│   └──────────────────────────────────────┘                     │
│   结果: T(n) = Θ(n^(log_b a))                                  │
│                                                                │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│   情况2: f(n) 与 n^(log_b a) 同阶                              │
│   ┌──────────────────────────────────────┐                     │
│   │    n^(log_b a)·log^k n               │ ← f(n)              │
│   │    ████████████████████████           │                     │
│   │                                     ████                   │
│   └──────────────────────────────────────┘                     │
│   结果: T(n) = Θ(n^(log_b a)·log^(k+1) n)                     │
│                                                                │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│   情况3: f(n) 多项式大于 n^(log_b a)                           │
│   ┌──────────────────────────────────────┐                     │
│   │                              f(n)    │ ← 分解+合并成本     │
│   │                                     ████████████████████████│
│   │    n^(log_b a)                       │                     │
│   │    ████                              │                     │
│   └──────────────────────────────────────┘                     │
│   结果: T(n) = Θ(f(n))                                        │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

### 📝 常见应用示例

#### 示例1：归并排序 T(n) = 2T(n/2) + Θ(n)

```
参数：
- a = 2（2个子问题）
- b = 2（规模减半）
- n^(log_b a) = n^(log_2 2) = n^1 = n
- f(n) = Θ(n)

比较：
- f(n) = Θ(n) 与 n 同阶
- 属于情况2（k = 0）

结论：T(n) = Θ(n^1 · log^1 n) = Θ(n log n) ✓
```

#### 示例2：T(n) = 3T(n/2) + n²

```
参数：
- a = 3
- b = 2
- n^(log_b a) = n^(log_2 3) ≈ n^1.585
- f(n) = n²

比较：
- f(n) = n² vs n^1.585
- n² 是多项式更大的（指数更大）
- 所以 f(n) = Ω(n^(log_2 3 + ε))，其中 ε ≈ 0.415

结论：属于情况3，T(n) = Θ(n²) ✓
```

#### 示例3：T(n) = 2T(n/2) + n²

```
参数：
- a = 2
- b = 2
- n^(log_b a) = n
- f(n) = n²

比较：
- f(n) = n² = Ω(n^(1+ε))，其中 ε = 0.5
- 检查正则条件：2·(n/2)² = 2·n²/4 = n²/2 ≤ c·n²（取 c = 1 即可）

结论：属于情况3，T(n) = Θ(n²) ✓
```

### ⚠️ 主方法的局限性

| 局限 | 说明 | 解决方法 |
|:---|:---|:---|
| **必须有 T(n) = aT(n/b)** | 必须是等分 | 情况不满足时用代入法 |
| **f(n) 必须是多项式** | 指数对数混合时不好用 | 用递归树法 |
| **情况2的 log^k** | k 必须是常数 | 如果是 log log n 这种，不适用 |

## 4.4 分治策略经典实例：归并排序

### 🎯 归并排序概述

归并排序（Merge Sort）是分治策略的完美示例，也是面试和竞赛中的常客。

### 🔄 算法思路图解

**排序 [8, 3, 1, 7, 4, 6, 2, 5]**

```
第1步：分解（Divide）
─────────────────────
原始数组: [8, 3, 1, 7, 4, 6, 2, 5]
           ↙          ↘
      [8,3,1,7]    [4,6,2,5]
        ↙   ↘       ↙   ↘
     [8,3]  [1,7]  [4,6]  [2,5]
       ↙  ↘   ↙ ↘  ↙  ↘  ↙  ↘
     [8] [3] [1] [7] [4] [6] [2] [5]  ← 8个单个元素（基本问题）


第2步：解决（Conquer）- 递归排序每个子数组
────────────────────────────────────────────
     [3,8]    [1,7]    [4,6]    [2,5]


第3步：合并（Combine）- 逐层合并并排序
──────────────────────────────────────
     [3,8]    [1,7]    [4,6]    [2,5]
        ↘   ↙/         ↘   ↙/
      [1,3,7,8]        [2,4,5,6]
            ↘         ↙/
         [1,2,3,4,5,6,7,8]  ← 最终结果
```

### 📝 Python 实现（含逐行注释）

```python
def merge_sort(arr):
    """
    归并排序主函数
    采用分治策略：分解 → 递归排序 → 合并
    
    参数:
        arr: 待排序的列表
    返回:
        排序后的新列表
    """
    # ============ 递归终止条件 ============
    # 如果数组只有一个或零个元素，本身就是有序的
    # 这是递归的"基本情况"，必须要有，否则会无限递归
    if len(arr) <= 1:
        return arr
    
    # ============ 分解步骤 (Divide) ============
    # 找到中点，将数组分成左右两半
    mid = len(arr) // 2          # 整数除法，找到中间位置
    
    # 递归地对左半部分排序
    left = merge_sort(arr[:mid])
    # 递归地对右半部分排序
    right = merge_sort(arr[mid:])
    
    # ============ 合并步骤 (Combine) ============
    # 将两个已排序的数组合并成一个有序数组
    return merge(left, right)


def merge(left, right):
    """
    合并两个已排序数组
    
    参数:
        left: 已排序的左数组
        right: 已排序的右数组
    返回:
        合并后的有序数组
    
    核心思想：双指针法
    - 维护两个指针分别指向两个数组的开头
    - 每次比较两个指针指向的元素，把较小的放入结果
    - 移动较小元素对应的指针
    - 重复直到某个数组为空，然后把另一个数组剩余部分直接加入
    """
    result = []  # 存放合并结果
    
    # ============ 双指针合并过程 ============
    i = j = 0  # i 指向 left，j 指向 right
    
    # 当两个数组都没遍历完时，比较并取较小的
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            # left[i] 更小（或相等），取 left[i]
            result.append(left[i])
            i += 1  # left 指针右移
        else:
            # right[j] 更小，取 right[j]
            result.append(right[j])
            j += 1  # right 指针右移
    
    # ============ 处理剩余元素 ============
    # 到这里，至少有一个数组已经遍历完了
    # 直接把另一个数组剩余的部分全部加入结果
    # （这些元素本来就是有序的，所以可以直接加入）
    
    # 如果 left 还有剩余，全部加入
    result.extend(left[i:])
    
    # 如果 right 还有剩余，全部加入
    # （extend 会把列表中的每个元素逐个加入）
    result.extend(right[j:])
    
    return result
```

### 📝 C++ 实现（含逐行注释）

```cpp
#include <iostream>
#include <vector>
using namespace std;

/**
 * 合并两个已排序的子数组
 * 
 * 参数:
 *   arr: 目标数组
 *   left: 左边界（包含）
 *   mid: 中间位置（右边界的一半）
 *   right: 右边界（不包含）
 * 
 * 核心思想: 使用辅助数组，双指针合并
 */
void merge(vector<int>& arr, int left, int mid, int right) {
    // ============ 创建辅助数组 ============
    // 创建一个临时数组存储合并结果
    vector<int> temp(right - left);
    
    // ============ 双指针初始化 ============
    int i = left;      // 左半部分指针，指向当前位置
    int j = mid;       // 右半部分指针，指向当前位置
    int k = 0;         // temp 数组的写入位置
    
    // ============ 双指针合并过程 ============
    // 比较两个子数组的元素，按从小到大的顺序放入 temp
    while (i < mid && j < right) {
        if (arr[i] <= arr[j]) {
            // 左半部分的元素更小（或相等），取左半部分
            temp[k++] = arr[i++];  // 先赋值，再递增
        } else {
            // 右半部分的元素更小，取右半部分
            temp[k++] = arr[j++];
        }
    }
    
    // ============ 处理剩余元素 ============
    // 如果左半部分还有剩余，直接复制到 temp
    while (i < mid) {
        temp[k++] = arr[i++];
    }
    
    // 如果右半部分还有剩余，直接复制到 temp
    // 注意：这里不需要再处理右半部分
    // 因为 while 循环结束时，j >= right 意味着右半部分已经遍历完
    
    // ============ 复制回原数组 ============
    // 将合并后的结果从 temp 复制回 arr 的对应位置
    for (int p = 0; p < k; p++) {
        arr[left + p] = temp[p];
    }
}

/**
 * 归并排序主函数
 * 
 * 参数:
 *   arr: 待排序数组（引用，直接修改原数组）
 *   left: 排序范围左边界（包含）
 *   right: 排序范围右边界（不包含）
 * 
 * 分治步骤:
 *   1. 分解: 找到中点 mid
 *   2. 解决: 递归排序左右两半
 *   3. 合并: 将两个有序子数组合并
 */
void mergeSort(vector<int>& arr, int left, int right) {
    // ============ 递归终止条件 ============
    // 如果范围只有一个或零个元素，不需要排序
    // 这就是递归的"基本情况"
    if (right - left <= 1) {
        return;
    }
    
    // ============ 分解步骤 (Divide) ============
    // 找到中点
    int mid = left + (right - left) / 2;
    // 等价于: mid = (left + right) / 2
    // 但用 left + (right - left) / 2 可以防止整数溢出
    
    // ============ 解决步骤 (Conquer) ============
    // 递归排序左半部分 [left, mid)
    mergeSort(arr, left, mid);
    // 递归排序右半部分 [mid, right)
    mergeSort(arr, mid, right);
    
    // ============ 合并步骤 (Combine) ============
    // 将两个有序子数组合并
    merge(arr, left, mid, right);
}

// 包装函数，方便调用
void mergeSort(vector<int>& arr) {
    mergeSort(arr, 0, arr.size());
}

// ============ 测试代码 ============
int main() {
    // 测试用例
    vector<vector<int>> testCases = {
        {8, 3, 1, 7, 4, 6, 2, 5},
        {5, 2, 8, 1, 9, 3, 7, 4, 6},
        {1},
        {},
        {3, 3, 3, 1, 1, 2, 2}
    };
    
    for (const auto& arr : testCases) {
        cout << "原数组: ";
        for (int num : arr) {
            cout << num << " ";
        }
        cout << endl;
        
        vector<int> sorted = arr;
        mergeSort(sorted);
        
        cout << "排序后: ";
        for (int num : sorted) {
            cout << num << " ";
        }
        cout << endl;
        cout << "--------------------" << endl;
    }
    
    return 0;
}
```

### 📊 复杂度分析

```
归并排序 T(n) = 2T(n/2) + Θ(n)

递归树分析：
───────────
第0层: 1个问题，  规模n，    成本: n
第1层: 2个问题，  规模n/2，  成本: n/2 + n/2 = n
第2层: 4个问题，  规模n/4，  成本: n/4×4 = n
...
第i层: 2^i个问题，规模n/2^i，成本: 2^i × (n/2^i) = n
总共 log n + 1 层

总成本 = n × (log n + 1) = n log n + n = Θ(n log n)

空间复杂度:
- 递归调用栈: O(log n)
- 辅助数组: O(n)
- 总空间: O(n)
```

## 4.5 更多分治策略实例

### 📚 实例1：最大子数组问题

**问题**：给定一个整数数组，找到和最大的连续子数组。

**示例**：
```
输入: [-2, 1, -3, 4, -1, 2, 1, -5, 4]
输出: [4, -1, 2, 1]（和为6）
```

**分治解法思路**：

```
分解：把数组从中间分成两半
情况：
  1. 最大子数组在左半部分 → 递归解决
  2. 最大子数组在右半部分 → 递归解决
  3. 最大子数组跨越中点 → 单独计算（向左扩展+向右扩展）
合并：取三种情况中和最大的
```

**Python 实现**：

```python
def max_crossing_sum(arr, left, mid, right):
    """
    计算跨越中点的最大子数组和
    
    时间复杂度: O(n)
    """
    # 向左扩展：找到从中点向左的最大和
    left_sum = float('-inf')  # 负无穷
    current_sum = 0
    for i in range(mid, left - 1, -1):
        current_sum += arr[i]
        left_sum = max(left_sum, current_sum)
    
    # 向右扩展：找到从中点+1向右的最大和
    right_sum = float('-inf')
    current_sum = 0
    for i in range(mid + 1, right + 1):
        current_sum += arr[i]
        right_sum = max(right_sum, current_sum)
    
    # 跨越中点的最大和 = 左边最大 + 右边最大
    return left_sum + right_sum


def max_subarray_sum(arr, left, right):
    """
    分治法求最大子数组和
    
    递归式: T(n) = 2T(n/2) + O(n)
    总复杂度: O(n log n)
    """
    # 基本情况：只有一个元素
    if left == right:
        return arr[left]
    
    # 分解：找到中点
    mid = (left + right) // 2
    
    # 解决：三种情况
    left_max = max_subarray_sum(arr, left, mid)        # 情况1
    right_max = max_subarray_sum(arr, mid + 1, right)  # 情况2
    cross_max = max_crossing_sum(arr, left, mid, right)  # 情况3
    
    # 合并：取最大值
    return max(left_max, right_max, cross_max)


# 测试
arr = [-2, 1, -3, 4, -1, 2, 1, -5, 4]
print(f"最大子数组和: {max_subarray_sum(arr, 0, len(arr)-1)}")
```

### 📚 实例2：Strassen 矩阵乘法

**问题**：两个 n×n 矩阵相乘。

**传统方法**：Θ(n³)

**Strassen 方法**：Θ(n^(log₂7)) ≈ Θ(n^2.807)

**核心思想**：将矩阵分成4块，通过巧妙的7次乘法（而不是8次）来减少复杂度。

```
┌───┬───┐     ┌───┬───┐     ┌─────────────┬─────────────┐
│ A │ B │     │ E │ F │     │ AE+BG       │ AF+BH       │
├───┼───┤  ×  ├───┼───┤  =  │             │             │
│ C │ D │     │ G │ H │     │ CE+DG       │ CF+DH       │
└───┴───┘     └───┴───┘     └─────────────┴─────────────┘
```

Strassen 的7次乘法：
```
M1 = A(F-H)
M2 = (A+B)H
M3 = (C+D)E
M4 = D(G-E)
M5 = (A+D)(E+H)
M6 = (B-D)(G+H)
M7 = (A-C)(E+F)
```

## 4.6 分治策略的适用条件

### ✅ 适合分治的问题特征

| 特征 | 说明 | 例子 |
|:---|:---|:---|
| **可分解** | 问题可以分成若干相似的子问题 | 排序、搜索 |
| **子问题独立** | 子问题之间不需要共享资源 | 归并排序 |
| **子问题可解** | 子问题规模足够小时可直接求解 | 单个元素 |
| **可合并** | 子问题的解可以合并为原问题的解 | 排序的合并 |

### ❌ 不适合分治的情况

| 情况 | 问题 | 更好的方法 |
|:---|:---|:---|
| **子问题重叠** | 子问题有大量重复计算 | 动态规划 |
| **分解成本太高** | 分解和合并比直接解还慢 | 朴素方法 |
| **问题不独立** | 子问题需要共享信息 | 整体考虑 |

### 🤔 分治 vs 动态规划

```
┌─────────────────────────────────────────────────────────────┐
│                    分治 vs 动态规划                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   分治策略                    动态规划                      │
│   ─────────                   ─────────                     │
│   ✓ 子问题不重复              ✗ 子问题大量重复               │
│   ✓ 自顶向下自然递归          ✓ 自底向上消除递归             │
│   ✓ 实现简单                  ✓ 时间复杂度更优               │
│                                                             │
│   例: 归并排序                  例: 斐波那契数列               │
│   T(n) = 2T(n/2) + n          F(n) = F(n-1) + F(n-2)        │
│   每次都算新的子问题           子问题被重复计算多次           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 本章小结

### 📌 核心要点

1. **分治策略的三步**：分解（Divide）→ 解决（Conquer）→ 合并（Combine）

2. **递归式**：T(n) = aT(n/b) + f(n)

3. **递归式求解三法**：
   - 代入法：猜测 + 数学归纳法验证
   - 递归树法：画图直观理解
   - 主方法：直接套公式

4. **主方法三种情况**：
   - 情况1：Θ(n^(log_b a))
   - 情况2：Θ(n^(log_b a) · log^(k+1) n)
   - 情况3：Θ(f(n))

5. **经典例子**：归并排序 T(n) = 2T(n/2) + Θ(n) = Θ(n log n)

### ⚠️ 易错点

| 易错点 | 错误表现 | 正确理解 |
|:---|:---|:---|
| 递归终止条件 | 忘记写基本情况 | 必须有，否则无限递归 |
| 递归式推导 | 分不清 a、b、n/b | a是子问题数，b是缩小倍数 |
| 主方法边界 | 套错情况 | 先比较 n^(log_b a) 与 f(n) |
| 合并复杂度 | 忽略合并成本 | 合并也是 O(n) 起步 |

## FAQ（常见问题解答）

### Q1: 分治策略和递归有什么关系？
A: 分治策略通常用递归来实现，但不是所有递归都是分治。分治强调"分解"和"合并"两个关键步骤。

### Q2: 为什么归并排序比快速排序更稳定，但快速排序实际更快？
A: 归并排序稳定是因为相等的元素不会被交换位置。快速排序实际更快是因为：
   - 归并排序需要额外的 O(n) 空间
   - 归并排序有大量的数据移动（拷贝）
   - 快速排序的缓存命中率更高

### Q3: 主方法的"正则条件"是什么？为什么需要？
A: 正则条件 af(n/b) ≤ cf(n)（c < 1）是为了确保递归树中每层的成本递减。如果不满足，即使 f(n) 多项式大于 n^(log_b a)，也不能直接用情况3。

### Q4: 递归深度太深会怎样？
A: 会导致栈溢出（Stack Overflow）。Python 默认递归深度约1000层，C++ 取决于系统栈大小。如果要处理大规模数据，可以改用迭代实现（消除递归）。

### Q5: 分治策略能否并行化？
A: 可以！分治策略天然适合并行计算，因为子问题之间是独立的。现代的多核CPU、GPU计算都大量使用分治策略。例如归并排序可以轻松并行化。
