## 8.1 排序算法下界

### 📖 问题引入：比较排序的极限

前面学的归并排序、堆排序、快速排序都是**比较排序**——它们通过"比较两个元素的大小"来决定顺序。

**问题**：有没有算法能在O(n)时间内排序？或者比较排序有更快的方法？

### 🧠 决策树模型

**决策树**是一个完全二叉树，用来抽象描述比较排序的过程：

```
                    ┌─────────────────┐
                    │   a₁ vs a₂      │  ← 第一次比较
                    └────────┬────────┘
                     ┌───────┴───────┐
                     ▼               ▼
                a₁ ≤ a₂          a₁ > a₂
                     │               │
              ┌──────┴──────┐  ┌──────┴──────┐
              ▼             ▼  ▼             ▼
           a₃ vs a₁      a₃ vs a₂  a₃ vs a₁    a₃ vs a₂
              ...           ...      ...         ...
```

**决策树的三要素**：
1. **每个内部结点**代表一次比较 `aᵢ ≤ aⱼ`
2. **每个叶子结点**代表一种可能的排序结果
3. **从根到叶的路径**代表排序过程

### 📐 下界证明：一步步推导

**第一步：叶子结点数**

对n个不同的元素，排序结果有 **n! 种可能**（n个元素的全排列）。

每个叶子结点对应一种结果，所以决策树至少有 **n! 个叶子**。

```
n个元素 → n!种排列 → 决策树至少有n!个叶子
```

**第二步：树高度与叶子数的关系**

设决策树高度为 h（即最长的根到叶路径）。

一棵高度为 h 的二叉树，**最多有 2^h 个叶子**（每层最多翻倍）。

```
叶子数 ≤ 2^h
n! ≤ 2^h
```

**第三步：取对数求高度下界**

```
h ≥ log₂(n!)
```

根据**斯特林公式**：
```
n! ≈ √(2πn) × (n/e)ⁿ
log₂(n!) ≈ n·log₂(n) - n·log₂(e) + O(log n)
         = Ω(n log n)
```

**结论**：任何比较排序算法，在最坏情况下至少需要 **Ω(n log n)** 次比较。

```
┌─────────────────────────────────────────────────────────┐
│                   核心定理                               │
│                                                         │
│   任何比较排序算法，在最坏情况下都需要 Ω(n log n) 次比较 │
│                                                         │
│   证明工具：决策树模型 + 斯特林公式                      │
└─────────────────────────────────────────────────────────┘
```

### ❓ FAQ：为什么堆排序已经是O(n log n)了还不能更快？

因为堆排序已经达到了这个下界！Ω(n log n)是一个**紧致下界**，意思是：
- 不存在任何比较排序能比 O(n log n) 更快
- 堆排序和归并排序已经"触及天花板"了

### 🎯 突破下界的方法

既然比较排序有下界，想突破 O(n log n) 就必须**不用比较**！

三种线性时间排序都是**非比较排序**：
1. **计数排序**：统计每个值出现多少次
2. **基数排序**：按位数逐个排序
3. **桶排序**：分桶 + 桶内排序

## 8.2 计数排序（Counting Sort）

### 📖 算法思想

**一句话概括**：统计每个数出现几次，然后按顺序放好。

### 🏠 生活类比：班级点名

老师要点名统计成绩分布：

```
📋 成绩范围：0-100分
    ↓
老师准备一张表：[0][1][2][3]...[100]
    ↓
扫视全班，每个成绩对应格子+1
    ↓
最后按格子顺序念名字
```

### 📐 算法详解

**输入条件**：n个元素，每个都在 **[0, k]** 范围内

**核心数组**：
- `A[1..n]`：输入数组
- `B[1..n]`：输出数组（排序结果）
- `C[0..k]`：**辅助数组**，C[i] = "小于等于i的元素个数"

**伪代码**：

```python
COUNTING-SORT(A, B, k):
    # 创建并初始化计数数组
    let C[0..k] be a new array
    for i = 0 to k:
        C[i] = 0              # ① 初始化，全为0
    
    # 统计：数每个值出现几次
    for j = 1 to A.length:
        C[A[j]] = C[A[j]] + 1  # ② 统计次数
    
    # 累加：计算"小于等于i"的有多少个（前缀和）
    for i = 1 to k:
        C[i] = C[i] + C[i-1]   # ③ 前缀和
    
    # 放置：从后往前，确保稳定性
    for j = A.length downto 1:
        B[C[A[j]]] = A[j]      # ④ 放到正确位置
        C[A[j]] = C[A[j]] - 1  # ⑤ 计数减1
```

### 📊 图解过程：一步步来

**输入数组**：`A = [2, 5, 3, 0, 2, 3, 0, 3]`，k = 5

**步骤①：初始化计数数组**
```
C = [0, 0, 0, 0, 0, 0]
     0  1  2  3  4  5  ← 索引
```

**步骤②：统计次数**
```
扫描A[1]=2 → C[2]++
扫描A[2]=5 → C[5]++
扫描A[3]=3 → C[3]++
扫描A[4]=0 → C[0]++
扫描A[5]=2 → C[2]++
扫描A[6]=3 → C[3]++
扫描A[7]=0 → C[0]++
扫描A[8]=3 → C[3]++

C = [2, 0, 2, 3, 0, 1]
     0  1  2  3  4  5
     
含义：0出现2次，2出现2次，3出现3次，5出现1次
```

**步骤③：累加（前缀和）**
```
C[0] = 2（不变）
C[1] = 0 + 2 = 2    # ≤1的有2个
C[2] = 2 + 2 = 4    # ≤2的有4个
C[3] = 3 + 4 = 7    # ≤3的有7个
C[4] = 0 + 7 = 7    # ≤4的有7个
C[5] = 1 + 7 = 8    # ≤5的有8个

C = [2, 2, 4, 7, 7, 8]
     0  1  2  3  4  5
     
含义：≤0的有2个，≤2的有4个，≤3的有7个...
```

**步骤④⑤：从后往前放置**
```
从后往前扫描A：

j=8: A[8]=3, C[3]=7 → B[7]=3, C[3]=6
j=7: A[7]=0, C[0]=2 → B[2]=0, C[0]=1
j=6: A[6]=3, C[3]=6 → B[6]=3, C[3]=5
j=5: A[5]=2, C[2]=4 → B[4]=2, C[2]=3
j=4: A[4]=0, C[0]=1 → B[1]=0, C[0]=0
j=3: A[3]=3, C[3]=5 → B[5]=3, C[3]=4
j=2: A[2]=5, C[5]=8 → B[8]=5, C[5]=7
j=1: A[1]=2, C[2]=3 → B[3]=2, C[2]=2

B = [0, 0, 2, 2, 3, 3, 3, 5]  ✅ 排序完成！
```

### 🤔 为什么要从后往前扫描？

**保证稳定性！** 即相同值的元素保持原有相对顺序。

```
示例：A = [3₁, 1, 3₂]（下标区分两个3）

从后往前：
j=3: 放3₂到位置C[3]
j=2: 放1到位置C[1]
j=1: 放3₁到位置C[3]-1（3₁在3₂前面）✅

结果：[3₁, 3₂, 1] 保持了3₁在3₂前面的顺序
```

### 💻 代码实现

**Python 实现**：
```python
def counting_sort(arr):
    """
    计数排序
    时间复杂度：O(n + k)，当 k = O(n) 时为 O(n)
    空间复杂度：O(n + k)
    稳定性：✅ 稳定
    """
    if len(arr) <= 1:
        return arr
    
    # ① 找到范围
    max_val = max(arr)
    min_val = min(arr)
    k = max_val - min_val + 1  # 计数数组大小
    
    # ② 创建计数数组并统计
    count = [0] * k
    for val in arr:
        count[val - min_val] += 1  # 统计次数
    
    # ③ 前缀和：计算累积个数
    for i in range(1, k):
        count[i] += count[i - 1]
    
    # ④ 从后往前放置（保证稳定性）
    output = [0] * len(arr)
    for val in reversed(arr):
        idx = val - min_val
        pos = count[idx] - 1  # 位置减1
        output[pos] = val
        count[idx] -= 1  # 计数减1
    
    return output


# 测试
arr = [2, 5, 3, 0, 2, 3, 0, 3]
print(counting_sort(arr))  # [0, 0, 2, 2, 3, 3, 3, 5]
```

**C++ 实现**：
```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

/**
 * 计数排序
 * @param arr 输入数组（引用）
 * @param k 值域范围（最大值）
 */
void countingSort(vector<int>& arr, int k) {
    int n = arr.size();
    if (n <= 1) return;
    
    // ① 找最大值
    int max_val = *max_element(arr.begin(), arr.end());
    k = max_val + 1;  // 值域大小
    
    // ② 创建计数数组并统计
    vector<int> count(k, 0);  // 初始化全0
    for (int val : arr) {
        count[val]++;  // 统计次数
    }
    
    // ③ 前缀和
    for (int i = 1; i < k; i++) {
        count[i] += count[i - 1];
    }
    
    // ④ 从后往前放置（保证稳定性）
    vector<int> output(n);
    for (int i = n - 1; i >= 0; i--) {
        int val = arr[i];
        output[count[val] - 1] = val;  // 放到正确位置
        count[val]--;  // 计数减1
    }
    
    // ⑤ 复制回原数组
    arr = output;
}


// 测试
int main() {
    vector<int> arr = {2, 5, 3, 0, 2, 3, 0, 3};
    countingSort(arr, 6);
    
    for (int val : arr) {
        cout << val << " ";
    }
    cout << endl;  // 输出: 0 0 2 2 3 3 3 5
    
    return 0;
}
```

### 📈 复杂度分析

| 指标 | 复杂度 | 说明 |
|:--:|:--:|:--|
| 时间复杂度 | **O(n + k)** | 第一/三/四循环：Θ(k) + Θ(n) + Θ(k) + Θ(n) |
| 空间复杂度 | **O(n + k)** | 需要额外的输出数组B和计数数组C |
| 稳定性 | ✅ **稳定** | 从后往前扫描保证相同值相对顺序不变 |

**关键条件**：当 **k = O(n)** 时，时间复杂度为 **Θ(n)**，即线性时间！

## 8.3 基数排序（Radix Sort）

### 📖 算法思想

**一句话概括**：把数字拆成一位一位，按每位从低到高排序。

### 🏠 生活类比：按学号排序

学生学号 231045678：
```
┌────────────────────────────────────────────────┐
│  按最后一位排  →  按倒数第二位排  →  按第三位排  │
│       ↓                  ↓                 ↓    │
│   231045670          231045670           ...   │
│   231045671          231045671              ...│
│   231045672  (每次只关心一位，其他位不变)       │
│      ...                                         │
└────────────────────────────────────────────────┘
```

### 📐 算法详解

**两种实现方式**：
1. **LSD（最低位优先）**：从最低位开始排
2. **MSD（最高位优先）**：从最高位开始排

**教材使用 LSD**，因为实现更简单。

**伪代码（LSD）**：

```python
RADIX-SORT(A, d):
    """
    对n个d位数进行基数排序
    A: 输入数组
    d: 位数（如最大数是9999，则d=4）
    """
    for i = 1 to d:           # 从低位到高位，循环d次
        # 使用稳定的排序算法按第i位排序
        # 这里用计数排序作为稳定排序
        stable_sort(A, i)      # 按第i位排序
```

### 🤔 为什么要用稳定排序？

**保证正确性！** 考虑两个数：170 和 45

```
原始：  170, 45
按个位：170, 45（个位都是0）
按十位：  45, 170

如果十位排序不稳定，可能变成 170, 45（错误！）
稳定排序确保：按十位排序时，个位排序的结果不被破坏
```

### 💻 代码实现

**Python 实现**：

```python
def counting_sort_for_radix(arr, exp):
    """
    计数排序的变体，按指定位数排序
    exp: 10的幂次，表示当前处理哪一位
         exp=1 表示个位，exp=10 表示十位，exp=100 表示百位
    """
    n = len(arr)
    output = [0] * n
    count = [0] * 10  # 数字0-9
    
    # ① 统计当前位的数字出现次数
    for val in arr:
        digit = (val // exp) % 10  # 取出当前位
        count[digit] += 1
    
    # ② 前缀和
    for i in range(1, 10):
        count[i] += count[i - 1]
    
    # ③ 从后往前放置（保证稳定性）
    for i in range(n - 1, -1, -1):
        digit = (arr[i] // exp) % 10
        output[count[digit] - 1] = arr[i]
        count[digit] -= 1
    
    return output


def radix_sort(arr):
    """
    基数排序（LSD，最低有效位优先）
    时间复杂度：O(d × (n + k))，d是位数，k是基数（这里是10）
    空间复杂度：O(n)
    稳定性：✅ 稳定
    """
    if len(arr) <= 1:
        return arr
    
    # ① 找最大值确定位数
    max_val = max(arr)
    
    # ② 从低位到高位，逐位排序
    exp = 1  # 从个位开始
    while max_val // exp > 0:
        arr = counting_sort_for_radix(arr, exp)
        exp *= 10
    
    return arr


# 测试
arr = [170, 45, 75, 90, 802, 24, 2, 66]
print(radix_sort(arr))  # [2, 24, 45, 66, 75, 90, 170, 802]
```

**C++ 实现**：

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

/**
 * 按指定位数排序（计数排序变体）
 * @param arr 输入数组（引用）
 * @param exp 当前位数（1=个位，10=十位，100=百位...）
 */
void countingSortByDigit(vector<int>& arr, int exp) {
    int n = arr.size();
    vector<int> output(n);
    vector<int> count(10, 0);  // 0-9共10个桶
    
    // ① 统计当前位数字出现次数
    for (int val : arr) {
        int digit = (val / exp) % 10;  // 取当前位
        count[digit]++;
    }
    
    // ② 前缀和
    for (int i = 1; i < 10; i++) {
        count[i] += count[i - 1];
    }
    
    // ③ 从后往前放置
    for (int i = n - 1; i >= 0; i--) {
        int digit = (arr[i] / exp) % 10;
        output[count[digit] - 1] = arr[i];
        count[digit]--;
    }
    
    // ④ 复制回原数组
    arr = output;
}


/**
 * 基数排序（LSD）
 */
void radixSort(vector<int>& arr) {
    if (arr.size() <= 1) return;
    
    // 找最大值确定位数
    int max_val = *max_element(arr.begin(), arr.end());
    
    // 从低位到高位逐位排序
    for (int exp = 1; max_val / exp > 0; exp *= 10) {
        countingSortByDigit(arr, exp);
    }
}


// 测试

    vector<int> arr = {170, 45, 75, 90, 802, 24, 2, 66};
    radixSort(arr);
    
    for (int val : arr) {
        cout << val << " ";
    }
    cout << endl;  // 输出: 2 24 45 66 75 90 170 802
    
    return 0;
}
```

### 📈 复杂度分析

| 指标 | 复杂度 | 说明 |
|:--:|:--:|:--|
| 时间复杂度 | **O(d × (n + k))** | d位数，每位用计数排序 |
| 空间复杂度 | **O(n + k)** | 输出数组 + 计数数组 |
| 稳定性 | ✅ **稳定** | 依赖稳定的计数排序 |

**关键条件**：
- 当 d 是常数，k = O(n) 时，为 **Θ(n)**
- 当 n 个 b 位数，r = ⌊logₖ(n)⌋ 时，为 **Θ(bn / logₙ(n))**

### ❓ FAQ：基数排序 vs 快速排序，选哪个？

| 场景 | 推荐算法 | 原因 |
|:--|:--|:--|
| n大，位数d小（如排序100万个3位数） | 基数排序 | d(n+k) = O(n) |
| n大，位数d大（如排序1000万个10位数） | 快速排序 | d(n+k) 可能比n log n大 |
| 需要原地排序（内存紧张） | 快速排序 | 基数排序需要O(n)额外空间 |

## 8.4 桶排序（Bucket Sort）

### 📖 算法思想

**一句话概括**：把数据分到若干桶里，每个桶内单独排序，然后按顺序合并。

### 🏠 生活类比：图书馆按楼层放书

图书馆按楼层（桶）存放书籍：

```
📚 图书馆
    ↓
┌─────────────────────────────────┐
│  1楼（桶0）：[0.1~0.2米区域]    │
│  2楼（桶1）：[0.2~0.3米区域]    │
│  3楼（桶2）：[0.3~0.4米区域]    │
│  ...                            │
└─────────────────────────────────┘
    ↓
先确定书的"身高"（值），放到对应楼层
    ↓
每层楼内部按书名排序
    ↓
从1楼到最高楼依次取出
```

### 📐 算法详解

**输入条件**：n个均匀分布在 **[0, 1) 区间内的浮点数**

**核心思想**：
1. 将区间 [0,1) 分成 n 个等长子区间（桶）
2. 每个桶管理一段小范围的数据
3. 桶内用插入排序（数据量小，插入排序很快）
4. 按桶顺序合并所有元素

**伪代码**：

```python
BUCKET-SORT(A):
    """
    桶排序
    A: 输入数组，假设所有元素在 [0, 1) 区间内
    """
    n = A.length
    let B[0..n-1] be a new array  # n个桶
    
    # ① 初始化桶（空链表）
    for i = 0 to n-1:
        B[i] = empty list
    
    # ② 将每个元素放入对应桶
    for i = 1 to n:              # 教材习惯从1开始
        index = floor(n * A[i])   # 计算桶编号
        insert A[i] into B[index] # 放入桶中
    
    # ③ 对每个桶内部排序（用插入排序）
    for i = 0 to n-1:
        sort list B[i] using insertion sort
    
    # ④ 按顺序合并所有桶
    concatenate B[0], B[1], ..., B[n-1] together
```

**输入**：`A = [0.78, 0.17, 0.39, 0.26, 0.72, 0.94, 0.21, 0.12, 0.68, 0.23]`，n = 10

### 💻 代码实现

**Python 实现**：

```python
def bucket_sort(arr):
    """
    桶排序
    假设输入是 [0, 1) 区间的浮点数
    时间复杂度：平均 O(n)，最坏 O(n²)
    空间复杂度：O(n)
    稳定性：✅ 稳定（如果桶内排序稳定）
    """
    if len(arr) <= 1:
        return arr
    
    n = len(arr)
    
    # ① 创建n个空桶
    buckets = [[] for _ in range(n)]
    
    # ② 将元素分配到对应桶
    for val in arr:
        index = int(val * n)  # 计算桶编号
        buckets[index].append(val)
    
    # ③ 桶内排序（用插入排序）
    for bucket in buckets:
        insertion_sort(bucket)  # 见下方定义
    
    # ④ 合并所有桶
    result = []
    for bucket in buckets:
        result.extend(bucket)
    
    return result


def insertion_sort(arr):
    """插入排序（用于桶内排序）"""
    for i in range(1, len(arr)):
        key = arr[i]
        j = i - 1
        while j >= 0 and arr[j] > key:
            arr[j + 1] = arr[j]
            j -= 1
        arr[j + 1] = key


# 测试（扩展版本，支持任意浮点数）
def bucket_sort_general(arr):
    """
    通用桶排序（支持任意范围的浮点数）
    """
    if len(arr) <= 1:
        return arr
    
    # ① 找范围
    min_val = min(arr)
    max_val = max(arr)
    
    # 如果所有元素相同，直接返回
    if min_val == max_val:
        return arr
    
    # ② 创建桶
    n = len(arr)
    buckets = [[] for _ in range(n)]
    
    # ③ 分配元素
    for val in arr:
        # 归一化到 [0, 1) 区间
        normalized = (val - min_val) / (max_val - min_val)
        index = int(normalized * (n - 1))  # 避免边界问题
        buckets[index].append(val)
    
    # ④ 桶内排序
    for bucket in buckets:
        bucket.sort()  # Python内置排序更快
    
    # ⑤ 合并
    result = []
    for bucket in buckets:
        result.extend(bucket)
    
    return result


# 测试
arr = [0.78, 0.17, 0.39, 0.26, 0.72, 0.94, 0.21, 0.12, 0.68, 0.23]
print(bucket_sort(arr))
# [0.12, 0.17, 0.21, 0.23, 0.26, 0.39, 0.68, 0.72, 0.78, 0.94]
```

**C++ 实现**：

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
#include <cmath>
using namespace std;

/**
 * 桶排序
 * 假设输入是 [0, 1) 区间的浮点数
 */
void bucketSort(vector<double>& arr) {
    int n = arr.size();
    if (n <= 1) return;
    
    // ① 创建n个桶
    vector<vector<double>> buckets(n);
    
    // ② 将元素分配到对应桶
    for (double val : arr) {
        int index = static_cast<int>(val * n);  // 计算桶编号
        buckets[index].push_back(val);
    }
    
    // ③ 桶内排序
    for (auto& bucket : buckets) {
        sort(bucket.begin(), bucket.end());
    }
    
    // ④ 合并所有桶
    int idx = 0;
    for (const auto& bucket : buckets) {
        for (double val : bucket) {
            arr[idx++] = val;
        }
    }
}


/**
 * 通用桶排序（支持任意范围）
 */
void bucketSortGeneral(vector<double>& arr) {
    int n = arr.size();
    if (n <= 1) return;
    
    // 找范围
    double min_val = *min_element(arr.begin(), arr.end());
    double max_val = *max_element(arr.begin(), arr.end());
    
    if (min_val == max_val) return;  // 所有元素相同
    
    // 创建桶
    vector<vector<double>> buckets(n);
    
    // 分配元素
    for (double val : arr) {
        // 归一化到 [0, 1)
        double normalized = (val - min_val) / (max_val - min_val);
        int index = static_cast<int>(normalized * (n - 1));
        buckets[index].push_back(val);
    }
    
    // 桶内排序
    for (auto& bucket : buckets) {
        sort(bucket.begin(), bucket.end());
    }
    
    // 合并
    int idx = 0;
    for (const auto& bucket : buckets) {
        for (double val : bucket) {
            arr[idx++] = val;
        }
    }
}
```

### 📈 复杂度分析

| 指标 | 复杂度 | 说明 |
|:--:|:--:|:--|
| 平均时间 | **O(n)** | 均匀分布时，每个桶约1个元素 |
| 最坏时间 | **O(n²)** | 所有元素在同一个桶 |
| 空间 | **O(n)** | 需要n个桶 + 输出数组 |
| 稳定性 | ✅ **稳定** | 依赖桶内排序的稳定性 |

**期望时间复杂度推导**：

设 n 个元素均匀分布在 [0,1) 区间：
- 每个桶期望有 **1 个元素**
- 桶内插入排序时间：**O(1)**
- n 个桶总时间：**O(n)**

### ❓ FAQ：桶排序为什么要求"均匀分布"？

**不均匀的影响**：

```
均匀分布：每个桶≈1个元素 → O(1) × n = O(n)

不均匀分布（如正态分布）：
  大量元素集中在中间桶 → 该桶内有 n/2 个元素
  桶内排序 O(n²) → 整体退化！
```

**如何处理不均匀**？
- 调整桶的数量
- 使用更智能的分桶策略（如递归桶排序）
- 对于严重不均匀，考虑其他排序算法

## 本章小结

### 🎯 核心要点

| 概念 | 核心思想 | 关键条件 |
|:--|:--|:--|
| **比较排序下界** | Ω(n log n) | 任何基于比较的排序 |
| **计数排序** | 统计每个值出现次数 | 值范围 k = O(n) |
| **基数排序** | 按位逐个稳定排序 | 稳定的位排序算法 |
| **桶排序** | 分桶 + 桶内排序 | 数据均匀分布 |

### 🔄 三种线性排序对比

| 排序算法 | 时间复杂度 | 空间复杂度 | 稳定性 | 数据要求 |
|:--:|:--:|:--:|:--:|:--|
| 计数排序 | O(n + k) | O(n + k) | ✅ | 整数，范围小 |
| 基数排序 | O(d(n+k)) | O(n + k) | ✅ | 整数/字符串 |
| 桶排序 | O(n)（平均） | O(n) | ✅ | 均匀分布浮点数 |

### 💡 何时使用

```
想排序，但不确定用哪个？
    ↓
数据是整数，范围不大？
    → 计数排序 ✅

数据是整数，位数固定？
    → 基数排序 ✅

数据是[0,1)区间的浮点数，分布均匀？
    → 桶排序 ✅

以上都不满足？
    → 快速排序/归并排序（O(n log n)）
```

### ⚠️ 注意事项

1. **线性时间排序不是万能的**：需要满足特定条件
2. **计数排序需要额外空间**：O(n + k)
3. **基数排序的位数要选好**：太少→桶太多，太多→循环次数多
4. **桶排序依赖均匀性**：不均匀会退化成O(n²)
