## 本章导读

### 🎯 一句话概括
> **找第i小的数，别傻傻地先排序！有更聪明的方法——随机分治，直接锁定目标！**

### 🏠 生活类比：选秀节目找前三名

想象选秀节目有10000人参赛，导演想知道"谁是前三名"：

```
🎬 普通方法（先排序）：把10000人全部排名 → O(n log n)
    ↓
聪明方法（选择算法）：主持人随机分组PK
    - 每组5人，内部比出最强
    - 组与组之间比，只留最强的几个
    - 大大减少比较次数
    ↓

```

**关键思想**：不排序，只"找第i小的"，就能精确锁定目标！

## 9.1 最小值和最大值（Minimum and Maximum）

### 📖 问题描述

给定n个互不相同的数，找出：
- **最小值**：最小的那个
- **最大值**：最大的那个

### 🏠 生活类比：全班最高和最矮

班主任想知道全班最高和最矮是谁：
- **笨方法**：比完所有人，记录最高和最低 → 2n-2次比较
- **聪明方法**：让同学们两两配对比，高个子站一边，矮个子站另一边 → n-1次比较

```
原始队列：3, 7, 1, 9, 2, 8, 4, 6
              ↓
两两配对：  (3,7) (1,9) (2,8) (4,6)
            ↓      ↓     ↓     ↓
各组胜出：   7     9     8     6   → 找最大值
各组淘汰：   3     1     2     4   → 找最小值
```

### 📐 算法实现

**Python实现：**

```python
def min_max(arr):
    """
    同时找最小值和最大值
    时间复杂度：O(n)，比较次数 n-1 次（最优）
    """
    n = len(arr)
    if n == 0:
        return None, None
    
    # 只有一个元素
    if n == 1:
        return arr[0], arr[0]
    
    # 初始化：如果前两个元素不同，小的是min，大的是max
    if arr[0] < arr[1]:
        min_val = arr[0]
        max_val = arr[1]
        start = 2
    else:
        min_val = arr[1]
        max_val = arr[0]
        start = 2
    
    # 从第三个元素开始，每次取一对
    for i in range(start, n, 2):
        # 两个一组比较
        if arr[i] < arr[i + 1]:
            # arr[i]是小的，arr[i+1]是大的
            if arr[i] < min_val:
                min_val = arr[i]
            if arr[i + 1] > max_val:
                max_val = arr[i + 1]
        else:
            # arr[i]是大的，arr[i+1]是小的
            if arr[i + 1] < min_val:
                min_val = arr[i + 1]
            if arr[i] > max_val:
                max_val = arr[i]
    
    return min_val, max_val

# 使用示例
arr = [3, 7, 1, 9, 2, 8, 4, 6]
min_val, max_val = min_max(arr)
print(f"最小值: {min_val}, 最大值: {max_val}")  # 最小值: 1, 最大值: 9
```

**C++实现：**

```cpp
#include <iostream>
#include <vector>
#include <utility>
using namespace std;

// 同时找最小值和最大值
// 时间复杂度：O(n)，比较次数 3*floor(n/2) ≈ 1.5n 次
pair<int, int> min_max(vector<int>& arr) {
    int n = arr.size();
    if (n == 0) return {0, 0};
    if (n == 1) return {arr[0], arr[0]};
    
    // 初始化：比较前两个元素
    int min_val, max_val;
    if (arr[0] < arr[1]) {
        min_val = arr[0];
        max_val = arr[1];
    } else {
        min_val = arr[1];
        max_val = arr[0];
    }
    
    // 从第三个元素开始，两两一组
    for (int i = 2; i < n; i += 2) {
        // 比较当前组内的两个元素
        if (arr[i] < arr[i + 1]) {
            // arr[i]更小，arr[i+1]更大
            if (arr[i] < min_val) min_val = arr[i];
            if (arr[i + 1] > max_val) max_val = arr[i + 1];
        } else {
            // arr[i]更大，arr[i+1]更小
            if (arr[i + 1] < min_val) min_val = arr[i + 1];
            if (arr[i] > max_val) max_val = arr[i];
        }
    }
    
    return {min_val, max_val};
}

int main() {
    vector<int> arr = {3, 7, 1, 9, 2, 8, 4, 6};
    auto [min_val, max_val] = min_max(arr);
    cout << "最小值: " << min_val << ", 最大值: " << max_val << endl;
    return 0;
}
```

### 📊 比较次数分析

| 方法 | 比较次数 | 分析 |
|:--:|:--:|:--:|
| 分别找min和max | 2n-2 | 每个n-1次 |
| **两两配对法** | **3⌊n/2⌋** | 约1.5n次，**最优！** |

**推导过程**：
- n为偶数：n/2对，每对内部1次比较(共n/2次)，每对中大的与max比较(共n/2次)，小的与min比较(共n/2次) → **总共3n/2次**
- n为奇数：3⌊n/2⌋次（类似推导）

### ❓ FAQ：为什么两两配对能减少比较？

关键在于**减少重复比较**：
- 笨方法：每个数都要和当前min、当前max各比较一次 → 2n-2次
- 聪明方法：先在每对内部比较出大小，大的只需和max比，小的只需和min比 → 约1.5n次

## 9.2 期望为线性时间的选择算法（Randomized Selection）

### 📖 问题描述

**选择问题**：在n个互不相同的数中，找出第i小的元素（也称为第i个顺序统计量）。

### 🏠 生活类比：面试筛选

面试官要从100份简历中找出"综合排名前10"的：

```
🎬 普通方法：把100份全部排序 → O(n log n)
    ↓
随机快排法（Quickselect）：
    1. 随机选一份简历作为"标杆"
    2. 其他简历和标杆比较：比标杆好的放左边，差的放右边
    3. 看标杆落在第几位置：
       - 正好第10名 → 完成！
       - 第15名 → 只需在左边9个中继续找
       - 第5名 → 只需在右边5个中继续找
    ↓
    大大减少工作量！
```

### 📐 算法思想：基于快速排序的分区

Quickselect算法的核心思想：

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   快速排序：两边都递归 → O(n log n)                     │
│                                                         │
│   快速选择：只递归一边 → O(n)                           │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**关键洞察**：一旦知道一个元素在排序后的位置，就只需要在它所在的子数组中继续找！

### 📊 算法过程图解

假设数组：[13, 19, 9, 5, 12, 8, 7, 4, 21, 2, 6, 11]

**目标：找第8小的数（从小到大第8个）**

```
第1轮：随机选基准 9
       分区后：[5, 8, 7, 4, 2, 6] | 9 | [13, 19, 12, 11, 21]
       基准位置 = 7（基准是第7小的数）
       目标第8 > 第7 → 在右半边继续找

第2轮：在右半边[13, 19, 12, 11, 21]随机选基准 12
       分区后：[11] | 12 | [13, 19, 21]
       基准位置 = 7 + 1 = 8（第8小！）
       目标第8 == 第8 → 找到！答案是12
```

### 📐 代码实现

**Python实现（带详细注释）：**

```python
import random

def randomized_select(arr, p, r, i):
    """
    随机化选择算法 - 找第i小的元素
    
    参数:
        arr: 输入数组
        p: 子数组起始索引
        r: 子数组结束索引
        i: 目标排名（从1开始，即第i小的数）
    
    返回: 第i小的元素
    
    时间复杂度: 平均 O(n)，最坏 O(n²)
    """
    # 基本情况：只有一个元素
    if p == r:
        return arr[p]
    
    # 随机分区，返回基准位置q
    q = randomized_partition(arr, p, r)
    
    # k是基准在排序后的位置（从1开始计数）
    k = q - p + 1
    
    # 比较目标位置和基准位置
    if i == k:
        # 正好是基准，找到了！
        return arr[q]
    elif i < k:
        # 目标在左半边，只递归左半边
        return randomized_select(arr, p, q - 1, i)
    else:
        # 目标在右半边，递归右半边，排名要减去左边的数量
        return randomized_select(arr, q + 1, r, i - k)

def randomized_partition(arr, p, r):
    """
    随机分区 - 随机选择基准，避免最坏情况
    
    思路：随机选一个元素，和最后一个元素交换，
          然后用标准的Lomuto分区
    """
    # 【关键】随机选择基准，避免固定基准导致的退化
    rand_idx = random.randint(p, r)
    arr[rand_idx], arr[r] = arr[r], arr[rand_idx]  # 交换到末尾
    
    # Lomuto分区方案
    pivot = arr[r]  # 基准值
    i = p - 1       # i是小于基准的元素的边界
    
    for j in range(p, r):
        if arr[j] <= pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]  # 把小的换到前面
    
    # 最后把基准放到正确位置
    arr[i + 1], arr[r] = arr[r], arr[i + 1]
    
    return i + 1  # 返回基准的位置

# 使用示例
arr = [13, 19, 9, 5, 12, 8, 7, 4, 21, 2, 6, 11]
i = 8  # 找第8小的数

result = randomized_select(arr, 0, len(arr) - 1, i)
print(f"第{i}小的数是: {result}")  # 输出: 第8小的数是: 12
```

**C++实现（带详细注释）：**

```cpp
#include <iostream>
#include <vector>
#include <cstdlib>
#include <ctime>
using namespace std;

// 随机分区（Lomuto方案）
int randomized_partition(vector<int>& arr, int p, int r) {
    // 【关键】随机选择基准，避免最坏情况
    srand(time(NULL));
    int rand_idx = p + rand() % (r - p + 1);
    swap(arr[rand_idx], arr[r]);  // 交换到末尾
    
    int pivot = arr[r];  // 基准值
    int i = p - 1;       // i是小于基准的元素的边界
    
    for (int j = p; j < r; j++) {
        if (arr[j] <= pivot) {
            i++;
            swap(arr[i], arr[j]);
        }
    }
    
    // 把基准放到正确位置
    swap(arr[i + 1], arr[r]);
    return i + 1;
}

// 随机化选择算法
int randomized_select(vector<int>& arr, int p, int r, int i) {
    // 基本情况
    if (p == r) {
        return arr[p];
    }
    
    // 随机分区
    int q = randomized_partition(arr, p, r);
    
    // k是基准在排序后的位置（从1开始）
    int k = q - p + 1;
    
    // 比较目标位置和基准位置
    if (i == k) {
        return arr[q];  // 找到了
    } else if (i < k) {
        return randomized_select(arr, p, q - 1, i);  // 在左半边找
    } else {
        return randomized_select(arr, q + 1, r, i - k);  // 在右半边找
    }
}

int main() {
    srand(time(NULL));
    vector<int> arr = {13, 19, 9, 5, 12, 8, 7, 4, 21, 2, 6, 11};
    int i = 8;  // 找第8小的数
    
    int result = randomized_select(arr, 0, arr.size() - 1, i);
    cout << "第" << i << "小的数是: " << result << endl;
    
    return 0;
}
```

### 📊 复杂度分析

| 指标 | 分析 |
|:--:|:--|
| **平均时间** | **O(n)** |
| 最坏时间 | O(n²)（基准总是最差的情况） |
| 空间 | O(log n)（递归栈深度） |

**期望时间复杂度推导**：

```
设T(n)是处理n个元素的时间

每次分区后，基准位置k是均匀分布的（因为随机选择）

T(n) = T(max(k-1, n-k)) + O(n)

平均情况下，k服从均匀分布：
E[T(n)] = (1/n) * Σ E[T(max(k-1, n-k))] + O(n)
        = (2/n) * Σ(k=1 to n/2) E[T(k)] + O(n)

通过级数求和可证：E[T(n)] = O(n)
```

### ❓ FAQ：为什么Quickselect比先排序快？

**关键区别**：

| 算法 | 操作 | 时间复杂度 |
|:--:|:--:|:--:|
| 先排序 | 完全排序两边 | O(n log n) |
| Quickselect | 只递归一边 | O(n)（平均） |

Quickselect利用了**"知道位置就不必排序"**的信息——一旦知道基准在第k位，就只关心目标在左还是右，不关心其他元素的顺序！

### 🏠 什么时候用Quickselect？

- ✅ 需要找第i小的元素，且i不固定
- ✅ 数据量大，不需要完全排序
- ✅ 只需要Top-K个元素
- ❌ 需要频繁查询不同i → 先排序更划算

## 9.3 最坏情况为线性时间的选择算法（Deterministic Selection）

### 📖 问题提出

Randomized Select的平均时间是O(n)，但**最坏情况是O(n²)**。

**问**：能不能保证最坏情况也是O(n)？

### 🧠 关键思想：保证基准总是"够好"

**问题根源**：随机选择的基准可能很差（总是最大或最小）

**解决思路**：精心选择基准，保证基准不会太差！

### 📐 算法流程：五分化组法

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   第一步：把数组分成 n/5 组，每组5个元素（最后一组除外）│
│                                                         │
│   第二步：每组内部排序，找出中位数                       │
│                                                         │
│   第三步：递归找出这些中位数的中位数 x                   │
│                                                         │
│   第四步：用x作为基准分区                                │
│                                                         │
│   第五步：递归处理目标所在的那半边                        │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 📊 过程图解

```
 22, 15, 10, 18, 23, 14]
```

### 📐 关键引理：为什么x是"好基准"？

**引理**：设x是中位数的中位数，则**至少有一半的组**中，**至少有3个元素**大于等于x或小于等于x。

**证明图解**：

```

│  │   │ │   │ │   │ │   │ │   │     │   │                 │
│  │   │ │   │ │   │ │   │ │   │     │   │                 │
│  │ x │ │   │ │   │ │   │ │   │     │   │                 │
│  │   │ │   │ │   │ │   │ │   │     │   │                 │
│  │   │ │   │ │   │ │   │ │   │     │   │                 │
│  └───┘ └───┘ └───┘ └───┘ └───┘     └───┘                 │
│    ↑      ↑      ↑      ↑      ↑            ↑            │
│  中位数  中位数  中位数  中位数  中位数        中位数       │
```

**数量分析**：

| 区域 | 元素数量 | 说明 |
|:--:|:--:|:--|
| 右上角（≥ x） | ≥ 3 × ⌈n/10⌉ | 一半的组中至少3个元素 ≥ x |
| 左下角（≤ x） | ≥ 3 × ⌈n/10⌉ | 一半的组中至少3个元素 ≤ x |
| 包含x的组 | ≤ 5个 | x所在组 |

**结论**：无论目标在左半边还是右半边，最多递归处理 **7n/10** 个元素！

**Python实现（带详细注释）：**

```python
def deterministic_select(arr, p, r, i):
    """
    确定性选择算法 - 保证最坏情况O(n)
    
    核心思想：精心选择基准（不是随机的）
    
    参数:
        arr: 输入数组
        p: 子数组起始索引
        r: 子数组结束索引
        i: 目标排名（从1开始）
    
    返回: 第i小的元素
    """
    n = r - p + 1
    
    # 基本情况：元素很少，直接排序返回
    if n <= 5:
        # 对这5个（或更少）元素排序
        sub_arr = arr[p:r+1]
        sub_arr.sort()
        return sub_arr[i - 1]
    
    # ========== 核心：找出中位数的中位数 x ==========
    
    # 把元素分成 n/5 组
    num_groups = (n + 4) // 5  # 向上取整
    
    # 存储各组中位数
    medians = []
    
    for j in range(num_groups):
        group_start = p + j * 5
        group_end = min(group_start + 5, r + 1)
        
        # 提取这一组
        group = arr[group_start:group_end]
        
        # 排序并取中位数
        group.sort()
        median = group[len(group) // 2]
        medians.append(median)
    
    # 递归找出medians的中位数
    # 中位数的数量是 num_groups ≈ n/5
    if len(medians) <= 5:
        medians.sort()
        x = medians[len(medians) // 2]
    else:
        # 递归调用自己来找中位数的中位数
        x = deterministic_select(
            medians, 0, len(medians) - 1, 
            (len(medians) + 1) // 2
        )
    
    # ========== 用x作为基准分区 ==========
    q = partition_around_pivot(arr, p, r, x)
    
    # 计算基准的位置
    k = q - p + 1
    
    # 比较并递归
    if i == k:
        return arr[q]
    elif i < k:
        return deterministic_select(arr, p, q - 1, i)
    else:
        return deterministic_select(arr, q + 1, r, i - k)


def partition_around_pivot(arr, p, r, pivot):
    """
    用指定值作为基准进行分区
    
    返回基准在分区后的位置
    """
    # 找到基准的位置
    pivot_idx = None
    for idx in range(p, r + 1):
        if arr[idx] == pivot:
            pivot_idx = idx
            break
    
    # 把基准换到最后
    if pivot_idx is not None:
        arr[pivot_idx], arr[r] = arr[r], arr[pivot_idx]
    
    # Lomuto分区
    i = p - 1
    for j in range(p, r):
        if arr[j] <= pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]
    
    arr[i + 1], arr[r] = arr[r], arr[i + 1]
    
    return i + 1


# 使用示例
arr = [13, 19, 9, 5, 12, 8, 7, 4, 21, 2, 6, 11]
result = deterministic_select(arr, 0, len(arr) - 1, 5)
print(f"第5小的数是: {result}")  # 输出: 第5小的数是: 7
```

**C++实现（带详细注释）：**

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
#include <cmath>
using namespace std;

// 用指定基准进行分区
int partition_around_pivot(vector<int>& arr, int p, int r, int pivot) {
    // 找到基准的位置
    int pivot_idx = -1;
    for (int idx = p; idx <= r; idx++) {
        if (arr[idx] == pivot) {
            pivot_idx = idx;
            break;
        }
    }
    
    // 换到最后
    if (pivot_idx != -1) {
        swap(arr[pivot_idx], arr[r]);
    }
    
    // Lomuto分区
    int i = p - 1;
    for (int j = p; j < r; j++) {
        if (arr[j] <= pivot) {
            i++;
            swap(arr[i], arr[j]);
        }
    }
    
    swap(arr[i + 1], arr[r]);
    return i + 1;
}

// 确定性选择算法
int deterministic_select(vector<int>& arr, int p, int r, int i) {
    int n = r - p + 1;
    
    // 基本情况：元素很少
    if (n <= 5) {
        vector<int> sub(arr.begin() + p, arr.begin() + r + 1);
        sort(sub.begin(), sub.end());
        return sub[i - 1];
    }
    
    // 第一步：分成 n/5 组，找出每组中位数
    int num_groups = (n + 4) / 5;  // 向上取整
    vector<int> medians;
    
    for (int j = 0; j < num_groups; j++) {
        int group_start = p + j * 5;
        int group_end = min(group_start + 5, r + 1);
        
        // 提取并排序这一组
        vector<int> group(arr.begin() + group_start, arr.begin() + group_end);
        sort(group.begin(), group.end());
        
        // 取中位数
        medians.push_back(group[group.size() / 2]);
    }
    
    // 第二步：递归找出中位数的中位数
    int x;
    if (medians.size() <= 5) {
        sort(medians.begin(), medians.end());
        x = medians[medians.size() / 2];
    } else {
        x = deterministic_select(medians, 0, medians.size() - 1, 
                                (medians.size() + 1) / 2);
    }
    
    // 第三步：用x分区
    int q = partition_around_pivot(arr, p, r, x);
    int k = q - p + 1;  // 基准的位置
    
    // 第四步：比较并递归
    if (i == k) {
        return arr[q];
    } else if (i < k) {
        return deterministic_select(arr, p, q - 1, i);
    } else {
        return deterministic_select(arr, q + 1, r, i - k);
    }
}

int main() {
    vector<int> arr = {13, 19, 9, 5, 12, 8, 7, 4, 21, 2, 6, 11};
    cout << "第5小的数是: " << deterministic_select(arr, 0, arr.size() - 1, 5) << endl;
    return 0;
}
```

```
       ↑             ↑
    找中位数      分区后递归
     的中位数     （最多7n/10）

可以证明这个递归式的解是 T(n) = Θ(n)
```

### ❓ FAQ：为什么选5个一组？

**选择5的原因**：

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   如果每组g个元素：                                          │
│                                                             │
│   - 分组时间：O(n)（每组内部排序）                           │
│   - 递归求中位数：T(⌈n/g⌉)                                   │
│   - 分区后递归：T(αn)，其中α = 1 - (⌈n/(2g)⌉)              │
│                                                             │
│   要让T(n) = O(n)，需要：                                   │
│   T(⌈n/g⌉) + T(αn) + O(n) = O(n)                           │
│                                                             │
│   经过计算，g=5时效果最好（α≈0.7）                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 9.4 中位数的性质

### 📖 定义

**中位数**是按顺序排列后位于中间的元素：
- n为奇数：中位数是第 (n+1)/2 小的数
- n为偶数：中位数是第 n/2 小和第 n/2+1 小的数的平均值

### 🏠 生活类比：家庭收入中位数

想象一个社区的家庭年收入：

```
[5万, 8万, 10万, 12万, 15万, 20万, 50万, 100万]

平均收入 = (5+8+10+12+15+20+50+100)/8 = 27.5万 ← 被富豪拉高

中位数 = (12+15)/2 = 13.5万 ← 更能反映普通人水平
```

**为什么重要**：中位数比平均值更能反映"典型值"！

### 📐 中位数的用途

1. **统计分析**：抗噪声，不受极端值影响
2. **数据分割**：快速将数据分成两半
3. **近似优化**：某些问题可以用中位数近似
