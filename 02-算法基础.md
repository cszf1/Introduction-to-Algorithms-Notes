## 2.1 插入排序

### 🃏 生活类比：整理扑克牌

想象你手里有一堆扑克牌，需要把它们按从小到大的顺序整理好。你会怎么做？

**大多数人会这样操作：**
1. 左手为空，从桌上拿起第一张牌
2. 拿起第二张牌，和手里的牌比较，放到正确位置
3. 拿起第三张牌，从右向左比较，找到位置插入
4. 重复以上步骤，直到所有牌都整理好

**关键观察：** 
- 左手手里的牌始终是有序的
- 每次只处理一张新牌
- 新牌需要和手里的牌逐一比较，找到正确位置

**这正是插入排序的核心思想！**

---

### 📝 问题定义：排序问题

**输入：** n个数的序列 ⟨a₁, a₂, ..., aₙ⟩

**输出：** 输入序列的一个排列 ⟨a'₁, a'₂, ..., a'ₙ⟩，满足 a'₁ ≤ a'₂ ≤ ... ≤ a'ₙ

> 💡 这里的"数"在计算机中被称为**关键词（key）**，是用于排序的关键值。

### 📄 伪代码分析

```plaintext
INSERTION-SORT(A)
1  for j = 2 to A.length          ← 外层循环：遍历所有待排序元素
2      key = A[j]                  ← 取出当前要插入的元素
3      // Insert A[j] into the sorted sequence A[1..j-1].
4      i = j - 1                  ← 从已排序部分的最右边开始比较
5      while i > 0 and A[i] > key ← 找到插入位置（条件：A[i] > key）
6          A[i+1] = A[i]          ← 元素右移，给 key 腾出位置
7          i = i - 1              ← 继续向左比较
8      A[i + 1] = key             ← 把 key 插入到正确位置
```

**逐行解释：**

| 行号 | 代码 | 解释 |
|------|------|------|
| 1 | `for j = 2 to A.length` | j 从2开始（第一个"新牌"），遍历到最后一个元素 |
| 2 | `key = A[j]` | 把当前元素 A[j] 存到 key 中（"手里拿着的新牌"） |
| 3 | 注释 | 说明下一步要把 key 插入到已排序序列中 |
| 4 | `i = j - 1` | i 指向已排序部分的最后一个元素（最右边的牌） |
| 5 | `while i > 0 and A[i] > key` | 只要还没越界且当前元素大于 key，就继续左移 |
| 6 | `A[i+1] = A[i]` | 元素右移一位（就像扑克牌右移给新牌腾位置） |
| 7 | `i = i - 1` | 继续向左检查下一个元素 |
| 8 | `A[i + 1] = key` | 把 key 插入到找到的位置（i+1 因为 i-- 之后停在比 key 小的元素上）|

### 💻 Python 实现（带详细注释）

```python
def insertion_sort(A):
    """
    插入排序算法
    参数: A - 待排序的列表（会被原地修改）
    返回: None（原地排序）
    
    时间复杂度: 平均 O(n²), 最好 O(n), 最坏 O(n²)
    空间复杂度: O(1) - 原地排序
    """
    
    # 外层循环：从第二个元素开始（索引1），遍历到最后一个元素
    # 原因：第一个元素（索引0）本身就是有序的，不需要处理
    for j in range(1, len(A)):
        
        # 第2步：把当前要插入的元素存到 key 中
        # 为什么要存？因为后面的移动操作会覆盖 A[j]
        key = A[j]
        
        # 第3步：从已排序部分的最右边开始向左扫描
        # i 初始化为 j-1，即已排序部分的最后一个元素
        i = j - 1
        
        # 第5步：内层循环
        # 条件1：i >= 0，确保不越界
        # 条件2：A[i] > key，已排序部分的元素比 key 大
        #      - 如果 A[i] > key，说明 key 应该插到 A[i] 前面
        #      - 所以把 A[i] 右移，给 key 腾出位置
        while i >= 0 and A[i] > key:
            A[i + 1] = A[i]  # 右移一位
            i = i - 1        # 继续向左检查
        
        # 第8步：退出循环时，i 指向比 key 小的元素（或 i=-1）
        # 所以 key 应该插入到位置 i+1
        A[i + 1] = key
    
    return A


# ==================== 测试代码 ====================
if __name__ == "__main__":
    # 测试用例
    test_cases = [
        [5, 2, 4, 6, 1, 3],      # 教材示例
        [31, 41, 59, 26, 41, 58], # 思考题2.1-1
        [1, 2, 3, 4, 5],          # 最佳情况：已有序
        [5, 4, 3, 2, 1],          # 最坏情况：逆序
        [3],                      # 边界情况：单个元素
        [],                       # 边界情况：空数组
    ]
    
    print("插入排序测试")
    print("=" * 50)
    
    for arr in test_cases:
        original = arr.copy()
        print(f"原始数组: {original}")
        insertion_sort(arr)
        print(f"排序后:   {arr}")
        print("-" * 50)
```

### 💻 C++ 实现（带详细注释）

```cpp
#include <iostream>
#include <vector>
using namespace std;

/**
 * 插入排序算法
 * 
 * 参数: A - 待排序的向量（会被原地修改）
 * 返回: void（原地排序）
 * 
 * 时间复杂度: 平均 O(n²), 最好 O(n), 最坏 O(n²)
 * 空间复杂度: O(1) - 原地排序
 */
void insertion_sort(vector<int>& A) {
    // 外层循环：j 从 1 遍历到 n-1（数组的第二个元素到最后一个元素）
    // 原因：索引 0 的元素本身就是有序的，不需要处理
    for (int j = 1; j < (int)A.size(); j++) {
        
        // 第2步：把当前要插入的元素存到 key 中
        // 命名 key 是因为它就是"关键词"，要找到它的正确位置
        int key = A[j];
        
        // 第4步：从已排序部分的最右边开始向左扫描
        // i 初始化为 j-1，即已排序部分的最后一个元素的位置
        int i = j - 1;
        
        // 第5步：内层循环
        // 条件1：i >= 0，确保不越界（数组下标不能为负）
        // 条件2：A[i] > key，已排序部分的元素比 key 大
        //      - 如果 A[i] > key，说明 key 应该插到 A[i] 前面
        //      - 所以把 A[i] 右移，给 key 腾出位置
        while (i >= 0 && A[i] > key) {
            A[i + 1] = A[i];  // 右移一位
            i = i - 1;        // 继续向左检查
        }
        
        // 第8步：退出循环时，i 指向比 key 小的元素（或 i=-1）
        // 所以 key 应该插入到位置 i+1
        A[i + 1] = key;
    }
}

// ==================== 主函数：测试代码 ====================
int main() {
    // 测试用例
    vector<vector<int>> test_cases = {
        {5, 2, 4, 6, 1, 3},      // 教材示例
        {31, 41, 59, 26, 41, 58}, // 思考题2.1-1
        {1, 2, 3, 4, 5},          // 最佳情况：已有序
        {5, 4, 3, 2, 1},          // 最坏情况：逆序
        {3},                      // 边界情况：单个元素
        {},                       // 边界情况：空数组
    };
    
    cout << "插入排序测试" << endl;
    cout << "=" << 50 << endl;
    
    for (int case_num = 0; case_num < (int)test_cases.size(); case_num++) {
        vector<int> arr = test_cases[case_num];
        vector<int> original = arr;
        
        cout << "测试用例 " << case_num + 1 << ":" << endl;
        cout << "原始数组: ";
        for (int x : original) cout << x << " ";
        cout << endl;
        
        if (!arr.empty()) {
            insertion_sort(arr);
        }
        
        cout << "排序后:   ";
        for (int x : arr) cout << x << " ";
        cout << endl;
        cout << "-" << 50 << endl;
    }
    
    return 0;
}
```

### 🔄 循环不变式（Loop Invariant）

**什么是循环不变式？**

循环不变式是关于程序循环的一个断言，它在循环的每次迭代开始和结束时都为真。

**插入排序的循环不变式：**

> 在 for 循环（变量为 j）的每次迭代开始时，子数组 A[1..j-1] 由原来在 A[1..j-1] 中的元素组成，但已按序排列。

**类比扑克牌：** 每轮迭代开始时，你左手中的牌（A[1..j-1]）始终是有序的。

---

### ✅ 用循环不变式证明正确性

循环不变式需要证明三条性质：

#### 1. 初始化（Initialization）

> **在第一次循环迭代之前（当 j = 2 时），循环不变式成立。**

**证明：** 当 j = 2 时，子数组 A[1..j-1] = A[1..1] 只包含一个元素 A[1]。只有一个元素的数组当然是有序的（这是平凡的有序）。所以初始化成立。

**扑克牌类比：** 刚开始时你手里只有一张牌，当然是有序的。

---

#### 2. 保持（Maintenance）

> **如果循环的某次迭代之前循环不变式为真，那么下次迭代之前它仍为真。**

**证明（课本原文）：**
for 循环体的第 4-7 行（while 循环）将 A[j-1]、A[j-2]、A[j-3] 等向右移动一个位置，直到找到 A[j] 的适当位置。第 8 行将 A[j] 的值插入该位置。

此时，子数组 A[1..j] 由原来在 A[1..j] 中的元素组成，但已按序排列。对 for 循环的下一次迭代增加 j 将保持循环不变式。

**扑克牌类比：** 你从桌上拿起一张新牌（A[j]），在手中的有序牌堆中从右向左比较，找到合适的位置插入。插入后，你手中的牌仍然是有序的。

---

#### 3. 终止（Termination）

> **在循环终止时，循环不变式为我们提供一个有用的性质，证明算法正确。**

**证明：** 导致 for 循环终止的条件是 j > A.length = n。因为每次循环迭代 j 增加 1，所以最终有 j = n + 1。

将循环不变式中的 j 替换为 n + 1，得到：子数组 A[1..n] 由原来在 A[1..n] 中的元素组成，但已按序排列。

注意到 A[1..n] 就是整个数组，我们推断出整个数组已排序。**因此算法正确。** ∎

**扑克牌类比：** 当你把所有牌都从桌上拿到手里后，手里的牌就是完整的一手有序牌。排序完成！

---

### ⏱️ 插入排序的时间复杂度分析

#### 最坏情况

**什么时候最慢？** 当数组完全逆序时。

对于每个 j（从 2 到 n），内层 while 循环要比较 j-1 次。

总比较次数 = 1 + 2 + 3 + ... + (n-1) = n(n-1)/2 = Θ(n²)

**结论：** 最坏情况时间复杂度是 **Θ(n²)**

#### 最佳情况

**什么时候最快？** 当数组已经有序时。

对于每个 j，内层 while 循环只执行一次比较就退出（因为 A[i] > key 为假）。

总比较次数 = n-1 = Θ(n)

**结论：** 最佳情况时间复杂度是 **Θ(n)**

#### 平均情况

通常也是 Θ(n²)，虽然比最坏情况好一半。

### 🔍 插入排序的特点

| 特点 | 说明 |
|------|------|
| **原地排序** | 只需要 O(1) 的额外空间 |
| **稳定排序** | 相等元素的相对顺序不变 |
| **在线排序** | 可以边接收数据边排序 |
| **自适应** | 对接近有序的数组效率很高 |
| **简单** | 代码容易理解和实现 |

**适用场景：**
- 数据量较小（n ≤ 50）
- 数据基本有序
- 作为高级排序算法的子过程（如快速排序的递归基准情况）

### ⏲️ 运行时间分析

#### 分析 INSERTION-SORT 的运行时间

```python
INSERTION-SORT(A)
1  for j = 2 to A.length:           # 执行 n 次
2      key = A[j]                    # 执行 n-1 次
3      i = j - 1                     # 执行 n-1 次
4      while i > 0 and A[i] > key:   # 总执行次数取决于输入
5          A[i+1] = A[i]             # 总执行次数取决于输入
6          i = i - 1                # 总执行次数取决于输入
7      A[i + 1] = key                # 执行 n-1 次
```

**各行代码的执行代价：**

| 行号 | 操作 | 代价 |
|------|------|------|
| 1 | `for j = 2 to A.length` | c₁ × n |
| 2 | `key = A[j]` | c₂ × (n-1) |
| 3 | `i = j - 1` | c₃ × (n-1) |
| 4 | `while ...` | c₄ × Σⱼ tⱼ |
| 5 | `A[i+1] = A[i]` | c₅ × Σⱼ (tⱼ - 1) |
| 6 | `i = i - 1` | c₆ × Σⱼ (tⱼ - 1) |
| 7 | `A[i + 1] = key` | c₇ × (n-1) |

> **tⱼ** 表示第 j 次迭代时内层 while 循环执行的次数

**最坏情况下（完全逆序）：** tⱼ = j（内层循环执行 j-1 次比较，再做1次比较失败）

**Σⱼ (tⱼ - 1)** = 1 + 2 + ... + (n-1) = n(n-1)/2

**总代价：** Θ(n²)

---

### 📊 增长量级（Rate of Growth）

**定义：** 当输入规模 n 很大时，我们只关心运行时间增长最快的项（最高阶项），忽略低阶项和常数系数。

**例子：**

| 精确公式 | 增长量级（当 n 很大时） |
|----------|------------------------|
| 6n³ + 3n² + 5n + 12 | Θ(n³) |
| 100n + 500 | Θ(n) |
| ½n(n-1) | Θ(n²) |

**为什么忽略常数？** 因为不同机器、不同编译器下常数差异很大，而增长量级反映的是算法本身的本质特征。

---

### 🔍 分析中的常见问题

#### 问题 1：如何确定 tⱼ 的值？

tⱼ 是内层 while 循环在第 j 次迭代时执行的次数。

**关键观察：** 
- 最佳情况：tⱼ = 1（key 已经在正确位置，只需要比较一次）
- 最坏情况：tⱼ = j（key 是最小元素，需要比较 j 次，再比较1次失败）

**求和计算：**

```
Σⱼ₌₂ⁿ (tⱼ) 的值：

最坏情况：Σⱼ₌₂ⁿ j = 2 + 3 + ... + n = n(n+1)/2 - 1 = Θ(n²)
最佳情况：Σⱼ₌₂ⁿ 1 = n-1 = Θ(n)
```

#### 问题 2：为什么用 Θ 表示？

Θ-notation（渐进紧确界）同时表示上下界。

- Θ(n²) 意味着增长量级既是 O(n²) 又是 Ω(n²)
- 当输入规模足够大时，运行时间大约与 n² 成正比

### 📝 渐近记号总结

| 记号 | 含义 | 类比 |
|------|------|------|
| **Θ(g(n))** | 渐近紧确界 | "大约等于 g(n)" |
| **O(g(n))** | 渐近上界 | "小于等于 g(n)" |
| **Ω(g(n))** | 渐近下界 | "大于等于 g(n)" |
| **o(g(n))** | 非渐近紧确上界 | "严格小于 g(n)" |
| **ω(g(n))** | 非渐近紧确下界 | "严格大于 g(n)" |

> **直觉理解：**
> - O 像 ≤（小于等于）
> - Ω 像 ≥（大于等于）
> - Θ 像 =（等于）
> - o 像 <（严格小于）
> - ω 像 >（严格大于）

## 2.3 设计算法

### 💡 两种算法设计策略

#### 1. 增量法（Incremental Approach）

**思想：** 一步一步地构建解，每一步都在已有解的基础上添加一个新元素。

**代表算法：** 插入排序

**过程：** 
```
已排序 A[1..j-1] → 添加元素 A[j] → 得到 A[1..j]
已排序 A[1..j] → 添加元素 A[j+1] → 得到 A[1..j+1]
...
```

#### 2. 分治法（Divide and Conquer）

**思想：** 把大问题分解成小问题，解决小问题，然后合并小问题的解。

**代表算法：** 归并排序

**过程：**
```
大问题 → 分解 → 若干小问题
                  ↓
            递归解决小问题
                  ↓
            合并 → 大问题的解
```

---

### 🧩 分治法的三个步骤

| 步骤 | 说明 | 插入排序 | 归并排序 |
|------|------|----------|----------|
| **分解（Divide）** | 将原问题分成子问题 | 把 A[j] 作为新元素 | 把数组分成两半 |
| **解决（Conquer）** | 递归地解决子问题 | 在已排序部分找位置 | 递归排序两个子数组 |
| **合并（Combine）** | 合并子问题的解 | 插入 A[j] | 合并两个有序子数组 |

---

## 2.3.1 分治法详解

### 🎯 分治法思想

**核心思想：** 递归地将问题分解为更小的相同子问题，直到子问题小到可以简单直接地解决，然后合并子问题的解。

**三大步骤：**

1. **分解（Divide）：** 将原问题划分为若干规模更小的子问题
2. **解决（Conquer）：** 递归地解决子问题；如果子问题足够小，则直接求解
3. **合并（Combine）：** 将子问题的解合并为原问题的解

---

### 📚 归并排序（Merge Sort）——分治法的经典例子

#### 生活类比：整理两堆已排序的扑克牌

假设你有两堆扑克牌，每堆都是从小到大排列好的。现在要把它们合并成一堆有序的牌。

**方法：**
1. 同时看两堆牌的顶牌
2. 拿出较小的放到结果堆
3. 重复直到某一堆为空
4. 把剩下的那堆全部放到结果堆

### 💻 Python 实现

```python
def merge_sort(A, p=0, r=None):
    """
    归并排序
    参数: 
        A - 待排序数组
        p - 子数组起始索引（默认0）
        r - 子数组结束索引（默认None，即数组末尾）
    返回: 排序后的数组（原地修改）
    
    时间复杂度: O(n log n) - 所有情况
    空间复杂度: O(n) - 需要额外数组
    """
    if r is None:
        r = len(A) - 1
    
    # 基本情况：子数组只有一个元素（已排序）
    if p < r:
        # 第2步：分解 - 找到中点
        q = (p + r) // 2
        
        # 第3步：递归排序左半部分
        merge_sort(A, p, q)
        
        # 第4步：递归排序右半部分
        merge_sort(A, q + 1, r)
        
        # 第5步：合并两个有序子数组
        merge(A, p, q, r)
    
    return A


def merge(A, p, q, r):
    """
    合并两个有序子数组
    参数:
        A - 数组
        p - 左子数组起始索引
        q - 左子数组结束索引（也是分割点）
        r - 右子数组结束索引
    
    合并 A[p..q] 和 A[q+1..r] 到 A[p..r]
    """
    # 第6-7步：计算两个子数组的大小
    n1 = q - p + 1  # 左半部分元素个数
    n2 = r - q      # 右半部分元素个数
    
    # 第8步：创建辅助数组
    # 额外留一个位置存放哨兵（无穷大）
    L = [0] * (n1 + 1)
    R = [0] * (n2 + 1)
    
    # 第9-10步：复制左半部分到 L
    for i in range(n1):
        L[i] = A[p + i]
    
    # 第11-12步：复制右半部分到 R
    for j in range(n2):
        R[j] = A[q + 1 + j]
    
    # 第13-14步：设置哨兵（简化边界检查）
    # Python 的 float('inf') 表示正无穷
    L[n1] = float('inf')
    R[n2] = float('inf')
    
    # 第15-16步：初始化索引
    i = 0  # L 的索引
    j = 0  # R 的索引
    
    # 第17步：逐个元素放入原数组
    for k in range(p, r + 1):
        if L[i] <= R[j]:
            A[k] = L[i]
            i += 1
        else:
            A[k] = R[j]
            j += 1
```

```cpp
/**
 * 合并两个有序子数组
 * 将 A[p..q] 和 A[q+1..r] 合并到 A[p..r]
 */
void merge(vector<int>& A, int p, int q, int r) {
    // 第6-7步：计算两个子数组的大小
    int n1 = q - p + 1;  // 左半部分元素个数
    int n2 = r - q;      // 右半部分元素个数
    
    // 第8步：创建辅助数组（+1 存放哨兵）
    vector<int> L(n1 + 1);
    vector<int> R(n2 + 1);
    
    // 第9-10步：复制左半部分到 L
    for (int i = 0; i < n1; i++) {
        L[i] = A[p + i];
    }
    
    // 第11-12步：复制右半部分到 R
    for (int j = 0; j < n2; j++) {
        R[j] = A[q + 1 + j];
    }
    
    // 第13-14步：设置哨兵（无穷大）
    L[n1] = numeric_limits<int>::max();
    R[n2] = numeric_limits<int>::max();
    
    // 第15-16步：初始化索引
    int i = 0;  // L 的索引
    int j = 0;  // R 的索引
    
    // 第17步：逐个元素放入原数组
    for (int k = p; k <= r; k++) {
        if (L[i] <= R[j]) {
            A[k] = L[i];
            i++;
        } else {
            A[k] = R[j];
            j++;
        }
    }
}

        merge_sort(A, q + 1, r);
        
        // 第5步：合并两个有序子数组

    }
}
```

### ⏱️ 归并排序的时间复杂度

#### 递归式分析

归并排序的时间复杂度可以用递归式表示：

```
T(n) = 
    Θ(1)                     如果 n = 1（基本情况）
    2T(n/2) + Θ(n)          如果 n > 1（分解 + 合并）
```

**解释：**
- **2T(n/2)**：递归解决两个规模为 n/2 的子问题
- **Θ(n)**：合并操作需要线性时间

---

### 🌳 递归树分析

```
                    T(n)
                   /    \
             T(n/2)    T(n/2)
              /  \      /  \
         T(n/4) T(n/4) T(n/4) T(n/4)
           ...          ...
           
每层代价:   n        n        n        ...       (合并操作)
层数:       0        1        2        ...       log₂n

总代价 = n × log₂n = Θ(n log n)
```

**为什么是 log₂n 层？**
- 每层都将问题规模减半
- 从 n → n/2 → n/4 → ... → 1
- 需要 log₂n 步才能从 n 降到 1

| 特性 | 插入排序 | 归并排序 |
|------|----------|----------|
| **时间复杂度（最坏）** | Θ(n²) | Θ(n log n) |
| **时间复杂度（最好）** | Θ(n) | Θ(n log n) |
| **时间复杂度（平均）** | Θ(n²) | Θ(n log n) |
| **空间复杂度** | O(1) | O(n) |
| **排序方式** | 原地排序 | 非原地排序（需要额外空间） |
| **稳定性** | 稳定 | 稳定 |
| **递归方式** | 增量法 | 分治法 |
| **适用场景** | 小规模/接近有序 | 大规模/任意数据 |

```
差距约 10 倍！

n = 1024 时

差距约 100 倍！
```

## 2.3.2 分析分治算法

### 📐 主定理（Master Theorem）

对于形如 **T(n) = aT(n/b) + f(n)** 的递归式，主定理给出了直接的解：

```
T(n) = aT(n/b) + f(n)

其中：
- a ≥ 1 是子问题的个数
- b > 1 是每个子问题的规模缩减因子
- f(n) 是分解和合并的成本
```

**主定理：**

| 条件 | 结论 |
|------|------|
| 如果 f(n) = O(n^log_b(a-ε))，ε > 0 | T(n) = Θ(n^log_b(a)) |
| 如果 f(n) = Θ(n^log_b(a) × log^k(n))，k ≥ 0 | T(n) = Θ(n^log_b(a) × log^(k+1)(n)) |
| 如果 f(n) = Ω(n^log_b(a+ε))，ε > 0，且正则条件 | T(n) = Θ(f(n)) |

---

### 🎯 归并排序的主定理应用

归并排序的递归式：**T(n) = 2T(n/2) + Θ(n)**

- a = 2（两个子问题）
- b = 2（每个子问题规模减半）
- f(n) = Θ(n)（合并操作线性时间）

**计算 n^log_b(a)：**
```
n^log₂(2) = n^1 = n
```

**比较 f(n) 和 n^log_b(a)：**
- f(n) = Θ(n)
- n^log_b(a) = Θ(n)

**属于第二种情况（k = 0）：**
- f(n) = Θ(n^log_b(a) × log⁰(n)) = Θ(n)
- 因此 T(n) = Θ(n × log n)

```
第1轮 (j=2, key=41):  31  41  59  26  41  58  ← 41 ≥ 31，无需移动
        ↓
第2轮 (j=3, key=59):  31  41  59  26  41  58  ← 59 ≥ 41，无需移动
        ↓
第3轮 (j=4, key=26):  31  41  59  59  41  58  ← 59 > 26，右移
                     31  41  41  59  41  58  ← 41 > 26，右移
                     31  31  41  59  41  58  ← 31 > 26，右移
                     26  31  41  59  41  58  ← 插入26
        ↓
第4轮 (j=5, key=41):  26  31  41  59  59  58  ← 59 > 41，右移
                     26  31  41  41  59  58  ← 41 = 41（不移动，因为 A[i] > key）
        ↓
第5轮 (j=6, key=58):  26  31  41  41  59  59  ← 59 > 58，右移
最终:  26  31  41  41  58  59  ← 排序完成
```

### 2.1-2

**题目：** 重写 INSERTION-SORT，使其按非升序（而不是非降序）排序。

**解答：** 只需把 `A[i] > key` 改成 `A[i] < key`：

```python
def insertion_sort_descending(A):
    for j in range(1, len(A)):
        key = A[j]
        i = j - 1
        while i >= 0 and A[i] < key:  # 改成 < 
            A[i + 1] = A[i]
            i -= 1
        A[i + 1] = key
    return A
```

### 2.1-3

**题目：** 线性查找问题：输入 n 个数的一个序列 A 和一个值 v，输出下标 i（若 v 在 A 中）或 NIL（若不在）。

**解答：**

```python
def linear_search(A, v):
    """
    线性查找
    返回: v 在 A 中的下标，或 None（NIL）
    """
    for i in range(len(A)):
        if A[i] == v:
            return i
    return None  # NIL
```

**循环不变式：**
> 在 for 循环的每次迭代开始时，未找到 v，且 v 不在已扫描的 A[0..i-1] 中。

### 2.1-4

**题目：** 两个 n 位二进制整数相加，返回 (n+1) 位和。

**解答：**

```python
def add_binary(A, B):
    """
    二进制加法
    输入: 两个 n 位二进制数组（低位在前，如 5 = [1,0,1]）
    输出: (n+1) 位和数组
    """
    n = len(A)
    C = [0] * (n + 1)  # 结果数组
    carry = 0
    
    for i in range(n):
        total = A[i] + B[i] + carry
        C[i] = total % 2      # 当前位的和
        carry = total // 2     # 进位
    
    C[n] = carry  # 最高位（可能是进位）
    return C
```

## ❓ FAQ（常见问题）

### Q1: 插入排序为什么从 j=2 开始？

**A:** 因为 A[1] 本身就是一个已排序的子数组（单元素天然有序）。如果从 j=1 开始，内层循环会试图和 A[0] 比较，但数组下标从 1 开始（教材约定）。

---

### Q2: 循环不变式和数学归纳法有什么区别？

**A:** 本质相同，但终止性不同：
- 数学归纳法：归纳步无限延续
- 循环不变式：循环在有限次后终止，终止时的状态提供有用信息

---

### Q3: 为什么归并排序需要哨兵（∞）？

**A:** 哨兵简化了边界检查。如果没有哨兵，每次循环都要检查两个子数组是否都还有剩余元素。使用哨兵后，当一个子数组全部取完后，其哨兵保证它永远是"更大"的，从而另一个子数组会被优先取完。

---

### Q4: 为什么归并排序不是原地排序？

**A:** 因为合并操作需要额外的线性空间来存储两个子数组的副本。虽然可以优化（如原地合并），但会显著增加代码复杂度和运行时间。

---

### Q5: 递归深度太深会怎样？

**A:** 可能导致栈溢出（Stack Overflow）。Python 默认递归深度限制约 1000 层。对于超大规模数据，可以考虑：
1. 调整递归深度限制（`sys.setrecursionlimit`）
2. 使用迭代版归并排序（自底向上）
3. 切换到非递归排序算法
